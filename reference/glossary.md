# Glossary

Plain-language definitions, architect-oriented. Link the **first use** of a term
in any kata to its entry here. Grow this as katas are written. Keep alphabetical.

> Format: **Term** (layer/area) — one-line definition. *Where it's taught:* kata id.

- **3-way handshake** — The TCP connection-setup sequence: SYN (client proposes sequence number) → SYN-ACK (server acknowledges and proposes its own) → ACK (client confirms); only after this exchange does data flow.
- **802.1Q** — The IEEE standard defining VLAN tagging; inserts a 4-byte tag into an Ethernet frame carrying a 12-bit VLAN ID field (values 0–4095; usable VLAN IDs are 1–4094, with 0 and 4095 reserved), enabling trunk links to carry multiple VLANs on one physical connection.
- **AAL (Authenticator Assurance Level)** — A NIST SP 800-63B tiered classification (AAL1/2/3) of how strongly an authentication event proves identity — AAL1 is single-factor, AAL2 requires MFA (TOTP or hardware key), AAL3 requires a hardware-based cryptographic authenticator that is also verifier-impersonation (phishing) resistant; used to specify authentication strength requirements in security policy.
- **ABAC** (security/IAM) — Attribute-Based Access Control: access decided by attributes (role, dept, time, resource tags). *S07.*
- **access port** — A switch port that carries traffic for exactly one VLAN and strips/adds the 802.1Q tag invisibly to the end device; the normal port type for servers, PCs, and IP phones.
- **Access Token** — A short-lived credential (JWT or opaque) issued by an OAuth2 authorization server that authorizes a client to call a specific API with specific scopes; validated by the resource server, not the client.
- **Account Vending Machine** — The automated pipeline (typically AWS Control Tower's Account Factory) that provisions new cloud accounts pre-configured with baseline monitoring, guardrails, and network wiring — so teams get a compliant spoke in minutes rather than weeks.
- **ACME protocol** — Automatic Certificate Management Environment (RFC 8555) — the protocol Let's Encrypt and compatible CAs use to automate domain validation and certificate issuance; what cert-manager uses under the hood to issue and renew certs without human intervention.
- **acquirer (payment acquirer)** — The financial institution or gateway that processes card payment transactions on behalf of a merchant, routing them to the card networks; the correct architectural point to tokenize a PAN before it enters any downstream system.
- **Active Directory (AD DS)** — Microsoft's on-prem directory service implementing LDAP + Kerberos + DNS on Domain Controllers; the authoritative source of user identities, group membership, and machine policies in most enterprise and FSI environments.
- **Address overlap** — The condition where two or more network segments share the same IP prefix, making it impossible for a router to distinguish which segment a packet is destined for — the primary cause of M&A and hybrid integration failures.
- **adequacy decision** — A European Commission finding that a non-EU country provides an essentially equivalent level of data protection to the EU, allowing personal data to flow there without additional legal mechanisms (SCCs, BCRs); as of 2026, India does not have an adequacy decision, so GDPR transfers to India require SCCs or another listed mechanism.
- **Administrative distance (AD)** — A per-source trustworthiness score (0–255, lower = more preferred) that a router uses to break ties when two protocols know the same prefix; static routes (AD 1) silently override OSPF routes (AD 110) for the same destination.
- **alert fatigue** — The operational state where security staff receive more alerts than they can meaningfully triage, leading to suppression of findings or ignored pages; the primary failure mode of an untuned detection service and a top risk when SCC or GuardDuty is enabled without a tuning programme.
- **ALG (Application-Layer Gateway)** — A NAT helper that inspects and rewrites IP addresses embedded inside application payloads (e.g., FTP, SIP) so they survive address translation; adds complexity and is often disabled in cloud environments. N16.
- **Amazon Inspector v2** — AWS's managed continuous vulnerability-scanning service (replacing the original Inspector) that automatically scans EC2 instance OS packages, Lambda function code, and ECR container images for CVEs, providing CWPP-tier findings without requiring a separate scan agent on each instance.
- **Anycast** — An IP addressing scheme where the same IP prefix is advertised from multiple geographic locations; the network routes each client to the nearest one — used by GCP's global HTTPS Load Balancing so a single VIP is served from all edge PoPs.
- **AoC (Attestation of Compliance)** — A formal document, signed by a Qualified Security Assessor (QSA) or the organisation itself, declaring that a defined scope of systems meets PCI-DSS requirements; the primary evidence a bank demands from Tier-1 payment vendors and cloud providers in lieu of an on-site audit.
- **API gateway** — A reverse proxy that adds API-management controls on top of load balancing: AuthN/AuthZ enforcement (JWT/API-key validation), rate limiting, request/response transformation, versioning, and per-route observability — enforced at one central point before requests reach backend services.
- **APIPA** — Automatic Private IP Addressing: a host self-assigns a 169.254.x.x/16 address when DHCP fails; a 169.254 address on a device is the reliable sign that DHCP did not respond.
- **AppSec (Application Security)** — The security function that owns SAST, DAST, dependency/SCA scanning, secure SDLC processes, and pre-production pen-testing of software; distinct from the SOC (which monitors runtime) and GRC (which governs policy).
- **ARP** (L2) — Address Resolution Protocol: finds the MAC for a known IP on the local segment. *N05.*
- **artifact provenance** — A signed statement (typically an in-toto attestation) recording how, when, and from which source revision a build artifact was produced — the evidence that lets a verifier confirm an artifact came from a trusted, uncompromised pipeline rather than an ad-hoc or attacker-controlled build.
- **ASN (AS Number)** — A unique number identifying an Autonomous System; 16-bit public ASNs (1–64495) are assigned by IANA/RIRs; 16-bit private ASNs are 64512–65534 (RFC 6996); a 32-bit private range 4200000000–4294967294 also exists; 0, 23456 and 65535 are reserved. Analogous to RFC 1918 for IPs.
- **Asymmetric routing** — A condition where the forward path (A→B) and return path (B→A) traverse different routers; breaks stateful firewalls that expect to see both directions of a flow. (L3)
- **attack surface** — The total set of paths — ports, API endpoints, user roles, input channels, management interfaces — through which a threat can interact with a system; shrinking it is the cheapest category of security control.
- **Authenticated encryption (AEAD)** — A symmetric encryption mode (e.g. AES-256-GCM) that simultaneously provides confidentiality and integrity in one operation, producing both ciphertext and an authentication tag; if any byte of ciphertext is altered, decryption fails rather than returning corrupted plaintext.
- **Authoritative name server** — The DNS server that holds the actual records for a zone and gives definitive (non-cached) answers; whoever owns the domain controls this.
- **Autonomous System (AS)** — A network under one administrative routing policy, identified by an ASN; the boundary between eBGP (inter-AS) and iBGP (intra-AS) sessions.
- **Availability Zone (AZ)** — A physically isolated data-center location within a cloud region; in AWS, subnets are scoped per-AZ, requiring one subnet per AZ for multi-AZ high availability.
- **AWS Firewall Manager** — AWS service for centrally authoring and enforcing WAF policies, Security Group baselines, and Network Firewall rules across multiple accounts in an AWS Organization — the AWS equivalent of GCP Hierarchical Firewall Policies.
- **AWS Network Firewall** — AWS's managed stateful NGFW deployed into a VPC subnet, supporting Suricata-compatible IDS/IPS rules, domain-based filtering, and traffic inspection for east-west and north-south VPC traffic — a workload-level complement to Security Groups and NACLs.
- **AWS RAM (Resource Access Manager)** — (AWS) An AWS service that allows a network account to share VPC subnets (and other resources) with other AWS accounts in the same organisation, enabling the AWS equivalent of GCP Shared VPC.
- **AWS Security Hub** — AWS's managed security findings aggregation service that collects, normalizes, and correlates findings from GuardDuty, Inspector, Macie, Config Rules, and third-party tools into a single pane, and maps them to compliance standards (CIS, PCI-DSS, NIST 800-53) — the AWS anchor for a CSPM capability.
- **AWS Shield** — AWS's managed DDoS protection service: Standard tier is always-on and free for all AWS customers; Advanced tier adds SLA, adaptive mitigation, and 24/7 DDoS response team access — the AWS equivalent of GCP Cloud Armor Managed Protection Plus.
- **AWS Verified Access** — AWS's per-application ZTNA service that gates access to internal applications based on identity (via an IdP) and device posture, removing the need for a client VPN and eliminating broad network-layer access.
- **AWS WAF** — AWS's web application firewall applied at CloudFront, Application Load Balancer, or API Gateway; inspects HTTP at L7 using custom and managed rule groups (e.g. OWASP CRS, bot-control); the AWS equivalent of GCP Cloud Armor's WAF functionality.
- **Azure Policy** — Azure's governance guardrail system applied at management group, subscription, or resource-group scope; can audit, deny, or auto-remediate resource configurations (e.g. block public IPs, require NSG flow logs) — the Azure equivalent of GCP Org Policy constraints.
- **Azure WAF** — Azure's web application firewall deployable on Azure Front Door or Azure Application Gateway; inspects HTTP/S for OWASP Top 10 attack classes using the OWASP CRS rule set — the Azure equivalent of GCP Cloud Armor's WAF function.
- **Backend service / backend pool** — The LB construct that groups a set of server instances (or endpoints) and carries the health-check, algorithm, and session-affinity config; called Backend Service in GCP and Target Group in AWS.
- **Bandwidth** — The maximum data rate a link or provisioned circuit can carry, in bits per second — a capacity figure, not a performance measurement. N53.
- **BCP (Business Continuity Plan)** — The organization-wide plan (covering people, process, and technology) that keeps critical business functions operating during and after an incident; broader than DR, which covers only IT systems.
- **BeyondCorp** — Google's internal Zero Trust access model (published 2014) that replaced VPN with identity- and device-verified proxied access to internal apps; the founding real-world ZT implementation and the model behind GCP BeyondCorp Enterprise.
- **BFD (Bidirectional Forwarding Detection)** — A lightweight protocol (RFC 5880) that sends fast hello packets between two endpoints to detect link or path failure in milliseconds, allowing routing protocols like OSPF or BGP to re-converge far faster than their native hello timers allow.
- **BGP** (L3) — Border Gateway Protocol: routing protocol that exchanges routes between networks; runs the internet and your cloud edge. *N14.*
- **Binary Authorization** — GCP's admission-control policy that requires container images to carry a cryptographic attestation (signed by a Cloud KMS key via an Attestor) before they can be deployed to GKE or Cloud Run; closes the supply-chain gap between a passing CI pipeline and what actually runs in production.
- **Blast radius** — how far an attacker or failure can spread from one compromised point; limited by segmentation. *N01, N27.*
- **blue team** — The defensive side of a SOC or security operations function: operates the SIEM, writes detection rules, triages alerts, and responds to incidents; contrasted with the red team that simulates adversaries.
- **BOLA / IDOR (Broken Object Level Authorization / Insecure Direct Object Reference)** — OWASP API Security Top 10 item #1: an API flaw where an authenticated caller can access any object by guessing its ID, because the backend does not verify that the caller owns or is permitted to access that specific object — the gateway can verify identity but cannot enforce per-object ownership.
- **Break-even (interconnect)** — The monthly data volume at which a committed dedicated interconnect port (flat fee + lower per-GB rate) becomes cheaper than routing the same traffic over an internet VPN or public egress; a key calculation before signing an interconnect commitment.
- **break-glass (change control)** — A pre-approved emergency change type that allows on-call teams to act immediately during an incident without waiting for a standard CAB window, subject to post-hoc documentation and review; the mechanism that reconciles FSI change-control culture with incident-response speed requirements.
- **break-glass access** — An emergency, audited path to a system that bypasses normal ZTNA or IAM controls — typically a PAM-vaulted credential with session recording and automatic revocation — used only when normal access fails; every use must generate an alert and a ticket.
- **break-glass account** — An emergency administrative account (typically Owner/root equivalent) held outside normal access controls and unlocked only in a declared incident; must be PAM-gated, session-recorded, and audited after every use. S07.
- **Broadcast address** — The last address in a subnet (all host bits set to 1); reaching every host in the subnet; not assignable to any device.
- **Broadcast domain** (L2) — the set of hosts an Ethernet broadcast (e.g. ARP) reaches; one switch/VLAN = one domain; routers and VLANs bound it. *N05, N15.*
- **Broken Access Control (A01)** — The #1 OWASP Top 10 risk: a design or implementation failure where an authenticated user can access resources or perform actions they are not authorised for — typically because authorisation is checked inconsistently per endpoint rather than enforced by a central policy; the most common FSI application security finding.
- **BYOH (Bring Your Own HSM)** — A cloud key-management model where the customer operates a dedicated, customer-managed HSM cluster (e.g. AWS CloudHSM, Azure Dedicated HSM) inside the provider's network, retaining full hardware custody — the highest compliance posture but also the highest operational burden.
- **BYOK (Bring Your Own Key)** — A cloud KMS model where the customer generates key material on-premises (ideally on an HSM) and imports it into the provider's KMS; gives the customer provenance over the key without requiring them to run their own HSM cluster in the cloud.
- **CAB** (process) — Change Advisory Board: the body that authorizes network/ infra changes and their windows in regulated shops. *N02.*
- **Cache-hit ratio** — The proportion of CDN requests served from edge cache without contacting the origin; a ratio above 80–90 % significantly reduces origin load and egress cost. *N47.*
- **CASB (Cloud Access Security Broker)** — A security control that sits between users and SaaS applications, providing visibility into data flows, shadow IT discovery, DLP policy enforcement, and blocking unauthorized uploads to unsanctioned cloud services.
- **CBWFQ (Class-Based Weighted Fair Queuing)** — A queuing discipline that allocates a guaranteed bandwidth percentage to each traffic class; serves classes proportionally when the link is congested.
- **CDE** (compliance) — Cardholder Data Environment: the PCI-DSS-scoped segment holding card data; must be isolated. *N29, S18.*
- **CDN** — Content Delivery Network: a globally distributed set of edge nodes (PoPs) that cache content near users, reducing latency and origin load; terminates TLS but does not inspect payloads for attacks. N25.
- **CE (Customer Edge)** — The router at the customer's site that connects to the MPLS carrier; the CE-to-PE link is the demarcation between the customer's responsibility and the carrier's managed core.
- **certificate chain** — The ordered sequence of certificates (leaf + one or more intermediates) that a TLS server presents so a client can walk the trust path up to a root CA it already holds in its trust store.
- **Certificate Transparency (CT)** — Public, append-only logs (RFC 6962) where CAs must record every issued certificate; enables auditors and domain owners to detect misissued or rogue certs (searchable at crt.sh).
- **CGNAT** — Carrier-Grade NAT: NAT performed by ISPs using RFC 6598 shared address space (100.64.0.0/10) to share a pool of public IPs across many subscribers; not globally routable.
- **chain of custody** — The documented, unbroken record of who handled digital evidence, when, and what they did with it — any gap or unverified transfer makes the evidence legally inadmissible or regulatorily challengeable.
- **Chronicle (Google Security Operations)** — GCP's cloud-native SIEM and SOAR platform (formerly Chronicle, now Google Security Operations) that ingests security telemetry at petabyte scale with a flat pricing model; the GCP equivalent of Microsoft Sentinel or a self-hosted Splunk/QRadar stack.
- **CIA triad** (security) — Confidentiality, Integrity, Availability: the three properties security protects. *S01.*
- **CIDR** (L3) — Classless Inter-Domain Routing: `a.b.c.d/n` notation; `/n` = number of network bits. *N08.*
- **CIEM (Cloud Infrastructure Entitlement Management)** — A discipline and tooling category that discovers, analyzes, and right-sizes cloud identity entitlements — human accounts and service accounts — to detect and remediate over-permissioned roles before they become attack paths; often embedded in CNAPP platforms. Pairs with S33.
- **CIR/EIR** — Committed Information Rate / Excess Information Rate: the bandwidth tiers in an MPLS or carrier WAN contract; CIR traffic is guaranteed delivery, EIR traffic is delivered best-effort and may be dropped during congestion.
- **CIS Controls** — The Center for Internet Security's 18 prioritised security controls (v8), organised into three Implementation Groups (IG1 basic hygiene, IG2 sensitive data, IG3 regulated/high-value); complemented by CIS Benchmarks — platform-specific hardening guides for operating systems and cloud services.
- **CISO (Chief Information Security Officer)** — The executive who owns an organisation's risk posture, security policy, control framework, and regulator/auditor relationship; typically reports to the CIO or directly to the board — board-level reporting signals a real design veto.
- **Classful addressing** — The pre-CIDR (pre-1993) scheme that divided IPv4 space into fixed Class A (/8), B (/16), and C (/24) blocks; abandoned because it wasted addresses and was replaced by CIDR.
- **Cloud Armor** — GCP's managed WAF and DDoS mitigation service applied to external load balancers; the GCP equivalent of AWS WAF + Shield.
- **Cloud Audit Logs** — GCP's control-plane and data-plane event log stream, comprising four types: Admin Activity (API calls that mutate state, always on), Data Access (reads and data writes, off by default due to volume), System Event, and Policy Denied — the primary forensic and compliance evidence source in GCP.
- **Cloud Controls Matrix (CCM)** — A control framework published by the Cloud Security Alliance (CSA, v4 current) that maps security controls to cloud service models (IaaS/PaaS/SaaS) and to compliance standards (ISO 27001, PCI-DSS, NIST); widely used in FSI GRC to map shared-responsibility boundaries to specific audit requirements.
- **Cloud DNS inbound policy** — (GCP) A Cloud DNS resource that creates reserved forwarding IP addresses inside a VPC subnet so on-premises DNS servers have a stable target to which they can forward private-zone queries.
- **Cloud IDS** — GCP's managed intrusion-detection service powered by Palo Alto Networks threat signatures; uses Packet Mirroring internally so no sensor VM is required. N54.
- **Cloud Interconnect** — GCP's dedicated private physical link (10 or 100 Gbps) between an on-prem router and a GCP VPC, carrying BGP-learned routes via Cloud Router; the GCP equivalent of AWS Direct Connect.
- **Cloud IPAM** — A service or process for tracking and allocating IP address ranges across cloud VPCs and on-prem, preventing CIDR overlap before it blocks hybrid connectivity.
- **Cloud NAT / NAT Gateway** — Managed outbound-only NAT service (GCP: Cloud NAT; AWS: NAT Gateway) that translates private VM IPs to one or more public IPs so VMs can initiate internet connections without being internet-addressable themselves. N41, N16.
- **Cloud Router** — GCP's managed BGP routing component that runs inside a VPC and exchanges routes with on-prem via Cloud Interconnect or HA VPN; the mechanism by which on-prem prefixes propagate into GCP spoke VPCs in a hub-and-spoke design.
- **CMDB** — Configuration Management Database: the authoritative registry of every network device, server, and service, including their relationships; used to validate that SNMP monitoring covers all known assets and that NetFlow anomalies can be attributed to known hosts.
- **CMEK (Customer-Managed Encryption Key)** — A key you create and control inside a cloud KMS (GCP Cloud KMS, AWS KMS, Azure Key Vault) that the cloud service uses to wrap its per-object DEKs; you can rotate or disable it to revoke the cloud provider's ability to decrypt your data, without holding key material outside the cloud boundary.
- **CNAPP (Cloud-Native Application Protection Platform)** — A Gartner term (coined 2021) for a unified security platform combining CSPM + CWPP + CIEM into a single risk graph capable of correlating configuration misconfigurations, workload vulnerabilities, and IAM entitlements into prioritized attack paths — fills the gap between point tools that surface individual findings.
- **conditional forwarding (conditional forwarder)** — A DNS resolver rule that forwards queries matching a specific domain suffix or zone to a designated nameserver rather than recursing to the public internet; the primary tool for bridging on-prem and cloud-private DNS namespaces so private cloud zones resolve from on-prem and vice versa.
- **Confidential computing** — A CPU-level capability (AMD SEV on GCP Confidential VMs, Intel TDX on Azure Confidential VMs, AWS Nitro Enclaves) that encrypts memory and protects data in use from the hypervisor and cloud operator, addressing the one state not covered by encryption-at-rest or in-transit controls.
- **CONNECT method (HTTP)** — The HTTP method a client sends to a proxy to open a raw TCP tunnel to a destination; used for all HTTPS traffic so the proxy relays the TLS stream without decrypting it. (N23)
- **Connected route** — A routing-table entry auto-created when an interface is configured with an IP address; marks the directly-attached subnet as reachable without a gateway. (L3)
- **Connection tracking table** — (firewall/L4) — a stateful firewall's in-memory record of active sessions (src IP, src port, dst IP, dst port, protocol, state); return packets matching an entry are automatically permitted without an explicit rule.
- **Connectivity Tests** — GCP Network Intelligence Center feature that simulates a packet through GCP routing and firewall configuration without sending real traffic, returning DELIVERED or DROPPED with the matching rule — used for pre-change validation and triage.
- **control plane** — The network paths and protocols that carry routing updates, BGP advertisements, DNS changes, and orchestration API calls — distinct from the data plane (application traffic) and the management plane (admin/SSH); a high-value attack target because poisoning a route or DNS record silently redirects all data-plane traffic.
- **Convergence (routing)** — The time for all routers to agree on an updated forwarding topology after a link or router change; OSPF convergence with default timers is ~40–90 s, tunable to sub-second with fast-hello timers and BFD.
- **copy-user anti-pattern** — Provisioning a new account by cloning the permissions of an existing peer rather than deriving them from the role definition; reliably propagates every accumulated excess permission the peer holds. S07.
- **cosign** — An open-source CLI tool (Sigstore project) for signing, verifying, and attaching attestations to OCI container images and artifacts, supporting both KMS-backed keys and keyless signing via short-lived Fulcio certificates recorded in the Rekor transparency log.
- **CPE (Customer Premises Equipment)** — The router or gateway device located at the customer's site that terminates a WAN circuit or interconnect and runs the BGP session on the enterprise side.
- **crash-consistent snapshot** — A cloud disk snapshot taken while a VM is running that captures a point-in-time state of the disk as if power were suddenly cut — file system metadata may be inconsistent but storage blocks are captured; contrast with application-consistent (quiesced writes before snapshot).
- **Credential stuffing** — An automated attack that replays username/password pairs leaked from one breach against other services, exploiting the widespread practice of password reuse; defeated by MFA (any type) but the attack volume makes bot-mitigation and rate-limiting equally important.
- **CRL (Certificate Revocation List)** — Signed list published by a CA (RFC 5280) of certificate serial numbers that have been revoked before their expiry; clients download it to check cert validity.
- **cross-connect** — A physical cable (copper or fiber) inside a colocation facility that links two different tenants' equipment — e.g., the cable between a GCP Cloud Interconnect port and an AWS Direct Connect port in the same Equinix cage; must be ordered separately from either cloud port.
- **Cryptoperiod** — The maximum time a given encryption key should remain in active use before rotation, as defined by NIST SP 800-57; PCI-DSS Req 3.7.4 mandates that cryptoperiods be defined and enforced for all keys protecting cardholder data.
- **CSPM (Cloud Security Posture Management)** — A tool or service that continuously scans cloud control-plane configuration (firewall rules, storage ACLs, encryption settings, logging gaps) against a policy baseline (CIS, PCI-DSS, custom) and reports findings — answers 'is anything misconfigured?'
- **CVSS (Common Vulnerability Scoring System)** — An open framework (scores 0–10) that quantifies the severity of a security vulnerability based on exploitability, impact, and scope; used to prioritize remediation: Critical (9–10) blocks deploys, High (7–8.9) fixes in-sprint, Medium/Low tracked on backlog.
- **CWPP (Cloud Workload Protection Platform)** — A tool or service that monitors risk inside cloud workloads (VMs, containers, serverless functions) — covering OS/package CVE scanning, runtime threat detection, and container image scanning — answers 'is this workload vulnerable or compromised?'
- **CycloneDX** — An OWASP-hosted SBOM standard (JSON/XML) widely adopted for its component-vulnerability linking and support for VEX (Vulnerability Exploitability eXchange) annotations; the format used by GCP Artifact Registry and AWS Inspector SBOM export.
- **DAST (Dynamic Application Security Testing)** — Black-box security testing that probes a running application over HTTP (or another protocol) to find runtime flaws — broken auth, injection, missing headers — that are invisible to static analysis; must run against a staging environment, never production.
- **data classification** — The act of assigning each data asset a sensitivity tier (Public / Internal / Confidential / Restricted) that drives downstream handling rules: access control, encryption, retention, audit logging, and DLP policy.
- **data fiduciary** — Under India's DPDP Act 2023, the entity that determines the purpose and means of processing personal data and bears primary regulatory liability for that data, regardless of which cloud or SaaS vendor stores it on their behalf.
- **data map** — A documented inventory of every data element an organization holds — what it is, where it lives, who owns it, and its classification tier; mandated by PCI-DSS Req 12.3.1 and RBI IT Framework §10 as the prerequisite for any DLP programme.
- **data plane** — The network paths that carry actual application traffic between users and services; distinguished from the control plane (routing/DNS signaling) and the management plane (administrative access); the primary focus of firewall rules, WAF policies, and segmentation controls.
- **dc3dd** — A DoD Cyber Crime Center fork of the Unix dd tool that adds built-in SHA-256 hashing, sector-by-sector error logging, and write-blocking support, making disk images produced by it forensically admissible.
- **DDI** — DNS + DHCP + IPAM: the integrated management discipline (and product category) that ties dynamic address assignment, hostname resolution, and address tracking into a single audit trail; Infoblox is the FSI gold standard.
- **DDoS** — Distributed Denial of Service — an attack that exhausts a resource (bandwidth, connection table, or application threads) using traffic from many sources; mitigation must occur upstream of the victim network.
- **Default gateway** (L3) — the router a host sends traffic to when the destination isn't on its local subnet. *N12.*
- **Default-deny** — (firewall/security) — a firewall posture where the final rule drops all traffic not explicitly permitted (whitelist model); mandated by PCI-DSS Req 1.3 and RBI guidelines for production FSI networks.
- **Defense in depth** (security) — layering independent controls so one failure isn't fatal. *S01.*
- **DEK (Data Encryption Key)** — A short-lived symmetric key an application generates to encrypt actual data; the DEK is itself encrypted by a KEK stored in a KMS, so the plaintext DEK never persists alongside the data it protects.
- **dependency confusion** — A supply-chain attack (Birsan, 2021) that exploits misconfigured build tools: by publishing a public package with the same name as an internal private package but a higher version number, an attacker causes the build resolver to pull their malicious package first.
- **Detection engineering** — The discipline of writing, testing, tuning, and versioning SIEM rules or ML models that convert raw log events into high-signal, low-noise security alerts mapped to known threat techniques — an ongoing engineering practice, not a one-time config.
- **DFD (Data Flow Diagram)** — A diagram of actors, processes, data stores, and data flows used as the primary input to a threat model; trust boundaries are drawn as dashed lines across which threats are enumerated.
- **DHCP** — Dynamic Host Configuration Protocol (RFC 2131): the protocol that automatically assigns IP address, subnet mask, default gateway, and DNS server to a host that broadcasts a request on joining a network.
- **DHCP relay agent** — A function on a router interface (e.g. Cisco 'ip helper-address') that intercepts DHCP broadcast DISCOVER messages and forwards them as unicast to a central DHCP server, enabling centralized DHCP across multiple subnets.
- **DHCP snooping** — An L2 switch feature that marks switch ports as trusted (DHCP server side) or untrusted (client side) and drops DHCP OFFER/ACK packets arriving on untrusted ports, blocking rogue DHCP servers.
- **DiffServ** — Differentiated Services: the IETF framework (RFC 2475) for scalable QoS using DSCP markings and per-hop behaviors rather than per-flow state in every router; the basis of modern enterprise and carrier QoS.
- **Direct Connect** — AWS's dedicated private network connection (1–100 Gbps) from an on-prem router to an AWS VPC, carrying BGP-advertised routes; the AWS equivalent of GCP Cloud Interconnect.
- **Direct Connect Gateway (AWS)** — An AWS construct that lets a single Direct Connect circuit reach VPCs across multiple AWS regions and accounts via Transit Gateways; enables hub-and-spoke hybrid connectivity without a circuit per region.
- **DLP (Data Loss Prevention)** — A control that inspects data at rest, in motion, or in use for policy violations (e.g. a card PAN in a public bucket) and takes action — alert, block, redact, or quarantine — based on classification tier and defined policies.
- **DMZ** (security/network) — a semi-trusted zone between the internet and internal networks for internet-facing services. *N27.*
- **DNS** (L7) — Domain Name System: resolves hostnames to IPs. *N17.*
- **DNS inbound endpoint** — A cloud-managed IP address inside your VPC (GCP) or a set of ENIs (AWS Route 53 Resolver) that accepts DNS queries from on-prem resolvers forwarded over a private link, enabling hybrid name resolution.
- **DNS outbound policy** — A VPC-level configuration (GCP: DNS outbound server policy; AWS: Route 53 Resolver outbound endpoint + forwarding rule) that instructs cloud resolvers to forward queries for specified zones to on-prem nameservers.
- **DNS record TTL (resource-record TTL)** — The number of seconds a recursive resolver may cache a DNS answer before re-querying the authoritative server; distinct from the IP-header TTL (hop counter in N06); low values enable fast failover, high values improve performance but slow propagation of changes.
- **DNSSEC** — DNS Security Extensions (RFC 4033–4035): cryptographic signing of DNS records so resolvers can verify answers have not been tampered with; protects against cache-poisoning attacks.
- **DORA** — The four-message DHCP handshake sequence: DISCOVER → OFFER → REQUEST → ACK; DISCOVER and REQUEST are client broadcasts because the client has no IP yet.
- **Double-extortion** — A ransomware attack pattern where the attacker exfiltrates sensitive data before encrypting it, then threatens both operational disruption and public data disclosure to increase leverage; means a successful restore does not eliminate the breach.
- **DPD (Dead Peer Detection)** — An IKEv2 keepalive mechanism that detects a silently dead VPN peer and triggers tunnel teardown and failover; prevents prolonged traffic blackholing.
- **DPDP Act** — Digital Personal Data Protection Act (India, 2023): the legislation governing how Indian personal data must be collected, processed, and stored; relevant to architects because it reinforces RBI data-residency obligations and imposes breach notification duties.
- **DPI (Deep Packet Inspection)** — Inspection of packet payloads (beyond headers) to identify the application generating traffic — used by SD-WAN CPEs and NGFWs to classify flows for app-aware routing or security policy. N33.
- **DPO (Data Protection Officer)** — A formally designated individual (required under GDPR Article 37 and DPDP equivalents for certain organisations) who advises on privacy obligations, monitors compliance, and acts as the point of contact for regulators; an architect involving personal data should loop in the DPO before finalising cloud region and data-flow design.
- **DSCP (Differentiated Services Code Point)** — A 6-bit field in the IP header used to mark packets for QoS treatment; routers and MPLS carriers honour DSCP markings to place traffic into the correct service class (e.g., voice vs best-effort).
- **Dual-stack** — A network host or infrastructure element that runs both IPv4 and IPv6 simultaneously, using whichever protocol the destination supports; the standard enterprise IPv6 migration approach.
- **dwell time** — The period between an attacker's initial compromise of a system and the organization's detection of the intrusion; longer dwell times increase blast radius and may corrupt backup windows.
- **Dynamic secrets** — Short-lived, on-demand credentials generated by a secrets engine (e.g. HashiCorp Vault database secrets engine) for each requesting workload and automatically revoked after a TTL; eliminates the need for long-lived shared credentials.
- **East-west traffic** — Lateral traffic between services or hosts inside a network (server-to-server, pod-to-pod); unchecked in flat networks and the primary vector for ransomware spread and attacker pivoting; controlled by internal firewalls, security groups, or micro-segmentation.
- **eBGP (external BGP)** — BGP sessions between routers in *different* Autonomous Systems — how ISPs, cloud providers, and enterprises exchange routes across administrative boundaries; the session type that 'runs the internet.'
- **ECMP** — Equal-Cost Multi-Path routing: distributes traffic across multiple equal-cost paths (e.g., all spines) by hashing flow identifiers, fully utilizing all links simultaneously without spanning-tree. (N30)
- **EF (Expedited Forwarding)** — DSCP value 46 (decimal); the highest-priority per-hop behavior defined in RFC 3246; used for voice/real-time traffic that cannot tolerate queuing delay or jitter.
- **Egress** (cloud/cost) — outbound data transfer; often the surprise line item in cloud bills. *N01, N51.*
- **Egress pattern** — One of four standard designs classifying how cloud VMs reach destinations outside the VPC: no egress, NAT-only egress, direct internet egress, or forced egress through a firewall/inspection appliance. N41.
- **Encapsulation** (all layers) — wrapping each layer's data in the next layer's header/trailer going down the stack; reversed (decapsulation) going up. *N04.*
- **ENI (Elastic Network Interface)** — AWS construct: the virtual NIC attached to an EC2 instance; Security Groups are attached to ENIs, not to the instance directly, so a multi-homed instance can have different rules per interface.
- **entitlement review** — A periodic, structured process (quarterly or annual) where a system owner certifies that every current access grant is still appropriate for the holder's current job function; the primary detective control against privilege accumulation. S07.
- **Entra ID** — Microsoft's cloud identity platform (formerly Azure AD) that extends Active Directory with SAML, OIDC, and SCIM connectors, enabling SSO federation to GCP, AWS, and third-party SaaS services from a single on-prem identity store.
- **Entra ID Connect** — The sync agent (formerly Azure AD Connect) that replicates on-prem Active Directory user accounts and group memberships into Entra ID, keeping cloud and on-prem identities in sync for federated SSO.
- **Envelope encryption** — The two-layer key pattern where a DEK encrypts data and a KEK encrypts the DEK; the encrypted DEK is stored alongside the ciphertext, while the KEK never leaves the KMS — used by Cloud KMS, AWS KMS, and every cloud-managed encryption service.
- **ephemeral port** — A short-lived source port (typically 32768–60999 on Linux, 49152–65535 per IANA; the Linux default differs from the IANA-recommended range) assigned automatically by the OS when a client opens a connection; firewall rules reference destination ports, not ephemeral ports.
- **EPSS (Exploit Prediction Scoring System)** — A probability score (0–1, updated daily by FIRST) estimating the likelihood that a specific CVE will have working exploit code in use within the next 30 days; layered on top of CVSS to separate theoretical risk from imminent threat.
- **ESP (Encapsulating Security Payload)** — The IPsec protocol that encrypts and authenticates the packet payload; uses IP protocol number 50; the component that carries actual data through a VPN tunnel.
- **EUI-64** — A method for deriving a 64-bit IPv6 Interface Identifier from a 48-bit MAC address by inserting 0xFFFE in the middle and flipping bit 7; used by SLAAC to form a host's interface portion of its address.
- **FAIR (Factor Analysis of Information Risk)** — A quantitative risk framework that converts qualitative likelihood and impact scores into expected annual dollar loss, used when the board requires financial framing rather than High/Medium/Low heat maps.
- **FHRP** — First-hop redundancy protocol — a family of protocols (HSRP, VRRP) that give two or more routers a shared virtual IP/MAC so hosts have a fault-tolerant default gateway.
- **FIDO2** — The FIDO Alliance open standard (combining W3C WebAuthn + CTAP2) for public-key-based authentication where a key pair is generated per site, the private key never leaves the device, and the signed challenge is cryptographically bound to the relying-party domain — making credentials inherently phishing-resistant.
- **FIPS 140-2 / FIPS 140-3** — US NIST standards for cryptographic module validation; Level 2 requires tamper-evidence; Level 3 requires tamper-resistance and identity-based authentication — the level that HSM-backed KMS keys and dedicated HSMs achieve, and the bar FSI auditors check against.
- **forensic image** — A bit-for-bit copy of a storage device (disk, volume, or RAM) taken without altering the original, verified with a cryptographic hash (SHA-256) at acquisition and at each transfer — distinct from a backup because it preserves deleted files, slack space, and is legally defensible.
- **Format-Preserving Encryption (FPE)** — A cipher mode (NIST SP 800-38G, algorithms FF1 and FF3-1) that produces a ciphertext in the same format as the plaintext — e.g. a 16-digit number that passes the Luhn check — allowing tokenization without schema changes in legacy systems.
- **Forward proxy** (L7) — sits in front of *clients*, controlling/observing their outbound traffic. *N23.*
- **Forward secrecy** — Property of a key-exchange algorithm (e.g. ECDHE) where session keys are ephemeral; compromise of the server's long-term private key cannot decrypt previously recorded sessions.
- **Forwarding rule (cloud LB)** — A GCP Cloud Load Balancing construct that maps a frontend IP+port to a backend service; billed by the hour per rule regardless of traffic, making unused rules a hidden cost.
- **Forwarding zone (DNS)** — A DNS zone configuration that proxies queries for a given domain to a designated upstream resolver (e.g., an on-prem DNS server) rather than answering authoritatively; used in GCP Cloud DNS and AWS Route 53 Resolver rules for hybrid name resolution.
- **Front door** — The combined CDN + WAF + global load balancer entry point for an internet-facing service; terminates TLS, caches static content at the edge, inspects traffic for attacks, and routes clean requests to the origin. N25.
- **FWaaS (Firewall-as-a-Service)** — A cloud-hosted L4/L7 firewall delivered from SASE/SSE PoPs that enforces egress security policy for branch offices and remote users without requiring a physical firewall appliance at each site.
- **Gateway VPC Endpoint** — A free, route-table-only AWS endpoint type (no ENI) supported exclusively for Amazon S3 and DynamoDB; adds a prefix-list route to the VPC route table rather than injecting an IP.
- **GCDS (Google Cloud Directory Sync)** — Google's sync tool that reads Active Directory or LDAP and populates Google Workspace / Cloud Identity with matching users and groups — the GCP-side equivalent of Entra ID Connect for Google-native deployments.
- **GDPR** — General Data Protection Regulation (EU, 2016/679): the EU law governing collection, processing, and transfer of personal data of EU residents; imposes lawful-basis requirements, 72-hour breach notification to the supervisory authority, and mechanisms (adequacy decisions, SCCs) for transferring data to non-EU countries — a cross-border transfer constraint architects must resolve before choosing a cloud region for EU-resident data.
- **giaddr** — Gateway IP Address — the field in a DHCP packet populated by a relay agent with the relay interface's IP; tells the DHCP server which subnet scope to draw from when responding.
- **Global front door** — A cloud load balancer that uses anycast to accept traffic at the nearest point of presence worldwide and route it to backends over the provider's private backbone, avoiding public internet hops.
- **GMEK (Google-Managed Encryption Key)** — GCP's default encryption model where Google generates, stores, and rotates the KEK on the customer's behalf with no customer visibility into the key; all storage is encrypted but the customer has no revocation lever — the provider can decrypt data using its own key. The AWS equivalent default is the "AWS managed key". Suitable for non-regulated data but not for PCI-scoped or RBI-regulated assets.
- **gMSA (group Managed Service Account)** — A Windows Active Directory account type for non-human workloads (Windows services, IIS app pools) where the domain automatically generates and rotates a complex password — the on-prem equivalent of GCP Workload Identity for Windows-hosted services.
- **Gratuitous ARP** — An ARP reply sent without a prior request, used by a newly-active FHRP router to announce that the virtual MAC now lives on its port, forcing switches to update their CAM tables immediately.
- **GRC (Governance, Risk & Compliance)** — The security function that owns the compliance evidence trail (PCI-DSS, RBI, ISO 27001), the risk register, the policy library, and third-party risk assessments; the function that produces documentation an auditor will examine.
- **GUA (Global Unicast Address)** — IPv6 addresses from the 2000::/3 block that are globally routable on the public internet; the IPv6 equivalent of public IPv4 addresses; assigned to cloud VPCs by providers.
- **GuardDuty** — AWS's managed threat-detection service that continuously analyzes CloudTrail management events, VPC Flow Logs, and DNS query logs using ML models and threat-intelligence feeds to produce severity-scored findings without requiring agents on instances.
- **Hairpin NAT** — When an internal host reaches a service by its public IP while both are inside the same network, forcing traffic to traverse the NAT device and back; fixed by split-horizon DNS so internal hosts resolve to the private IP directly. N16.
- **hard-fail (revocation)** — A TLS client configuration that aborts the connection when an OCSP responder is unreachable or returns an error, rather than soft-failing (allowing the connection through); required in high-assurance mTLS environments such as a bank's CDE.
- **Health check (LB)** — A periodic probe (TCP, HTTP, HTTPS) an LB sends to each backend to decide whether to include it in the active pool; governs how quickly a failed server stops receiving traffic.
- **Hierarchical Firewall Policy (GCP)** — GCP construct that enforces firewall rules at org, folder, or project scope, overriding VPC-level rules; the governance primitive that lets a central security team set hard boundaries no project team can bypass.
- **Host project** — (GCP Shared VPC) The GCP project that owns the Shared VPC, its subnets, and centrally managed firewall rules; the platform team's domain.
- **HSRP** — Hot Standby Router Protocol — Cisco-proprietary FHRP; routers elect an Active and a Standby; the Active owns the virtual IP/MAC; Standby takes over when hellos stop.
- **HYOK / External Key Manager (EKM)** — (Hold Your Own Key) The strictest key-custody model: key material never enters the cloud; every cloud encrypt/decrypt call phones out to the customer's own external key manager or HSM — highest assurance, highest operational risk if the external KMS is unavailable. GCP calls this Cloud EKM; AWS calls it KMS External Key Store (XKS).
- **IaaS / PaaS / SaaS** — The three NIST cloud service models (SP 800-145): Infrastructure-as-a-Service (customer owns OS and above), Platform-as-a-Service (customer owns app and data; provider owns OS and runtime), Software-as-a-Service (provider owns the application; customer owns data and access); each model shifts the shared-responsibility boundary between provider and tenant.
- **IAM** (security) — Identity & Access Management: managing who can do what. *S04–S08.*
- **IAM Access Analyzer** — An AWS service that continuously analyses IAM policies and resource-based policies to surface overly permissive access, cross-account exposure, and unused permissions; the AWS counterpart to GCP IAM Recommender.
- **IAM Identity Center (AWS)** — AWS's managed SSO and identity federation hub that connects an external IdP (e.g. Entra ID) to AWS accounts and permission sets, replacing per-account IAM users with centrally-managed, federated access.
- **IAM Policy Simulator** — A GCP (and AWS) tool that evaluates whether a given IAM principal would be allowed or denied a specific permission on a specific resource, based on the current policy bindings, without making a real API call.
- **IAM Recommender** — A GCP service that analyses 90 days of Cloud Audit Log data and recommends replacing over-broad IAM role bindings with the minimum set of permissions the principal actually used, quantifying the permission-reduction percentage.
- **IAST (Interactive Application Security Testing)** — A runtime security-analysis agent embedded in the application under test that observes actual code execution paths during functional testing, finding flaws with higher accuracy and fewer false positives than either SAST or DAST alone; requires instrumentation of the runtime (JVM, Node.js, etc.).
- **iBGP (internal BGP)** — BGP sessions between routers within the *same* Autonomous System, propagating externally-learned routes to interior routers without using OSPF to carry them.
- **ID Token** — A signed JWT issued by an OIDC IdP that identifies the authenticated user (claims: sub, iss, aud, exp); distinct from an Access Token, which authorizes API calls.
- **Identity Provider (IdP)** — The authoritative system that authenticates a user and issues signed assertions or tokens to Service Providers; holds the credentials and is the trust root in an SSO federation.
- **Identity-Aware Proxy (IAP)** — A reverse proxy that intercepts application requests and enforces identity authentication and optionally device posture before forwarding to the backend, implementing the ZTNA principle without requiring a client VPN; Google Cloud IAP is the canonical cloud-native example.
- **IDOR (Insecure Direct Object Reference)** — See **BOLA / IDOR**: an authorization vulnerability (OWASP Broken Access Control #1) where a service uses a user-supplied identifier — such as an account number in a URL — to access a resource without verifying that the authenticated caller is authorized to access that specific object.
- **IDS** — Intrusion Detection System — a passive sensor that receives a copy of traffic (via SPAN/mirror) and raises alerts on suspicious patterns; it cannot block traffic.
- **IKE (Internet Key Exchange)** — The protocol that negotiates cipher suites, performs Diffie-Hellman key exchange, and authenticates IPsec peers before data flows; runs on UDP 500/4500. IKEv2 (RFC 7296) is the current standard.
- **IMDSv2 (Instance Metadata Service v2)** — AWS's session-oriented, token-gated version of the EC2 metadata API (169.254.169.254) that requires a PUT pre-request to obtain a short-lived token before metadata can be read, preventing SSRF exploitation that simple GET requests (IMDSv1) allowed.
- **Immutable backup (Object Lock / WORM)** — A backup stored on write-once read-many (WORM) storage with a retention policy that prevents deletion or modification for a defined period, even by a compromised administrator; the key architectural control against ransomware destroying the recovery path.
- **Inbound resolver endpoint** — A real IP address provisioned inside a cloud VPC (GCP: inbound DNS policy; AWS: Route 53 Resolver inbound endpoint) that accepts DNS queries arriving over VPN or Interconnect from on-prem, forwarding them to the VPC resolver so private zone names can be resolved by on-prem clients.
- **indicator of compromise (IoC)** — An observable artifact — IP address, file hash, domain, registry key, URL, log pattern — whose presence on a system provides evidence that it has been compromised; shared via threat-intel feeds and used by SIEM and EDR to detect attacker presence. Valuable for blocking known-bad but short-lived because attackers rotate infrastructure; contrast with TTPs, which describe behavior.
- **Inherent risk** — The risk score (likelihood × impact) before any controls are applied; used to prioritize which risks demand treatment before the cost of controls is factored in.
- **Interface VPC Endpoint** — The AWS PrivateLink-based endpoint type: an ENI placed in a VPC subnet representing an AWS or third-party service; requires per-AZ provisioning and has hourly + data-processing costs.
- **intermediate CA (issuing CA)** — The online CA that signs end-entity (leaf) certificates day-to-day; it sits between the offline root and leaf certs so the root CA can revoke a compromised intermediate without itself being exposed or replaced.
- **internet exchange (IX)** — A physical colocation facility where multiple network providers (including cloud providers) peer or interconnect; used to establish private cross-cloud paths by co-locating Cloud Interconnect and Direct Connect ports in the same building.
- **Internet Gateway (IGW)** — AWS construct attached to a VPC that enables bidirectional internet traffic for instances with public IPs; a subnet is 'public' when its route table sends 0.0.0.0/0 to the IGW.
- **IPAM** — IP Address Management: the system or process (tool, spreadsheet, or dedicated software) an enterprise uses to track, allocate, and audit IP address space across sites and clouds.
- **iperf3** — Open-source active-measurement tool for measuring TCP and UDP throughput, jitter, and packet loss between two endpoints; the standard first test for 'how much capacity does this path actually deliver?'. N53.
- **IPFIX** — IP Flow Information Export (RFC 7011): the IETF-standard, template-based flow-export protocol that supersedes Cisco NetFlow v9; vendor-neutral and supports IPv6, MPLS, and custom fields.
- **IPS** — Intrusion Prevention System — an inline sensor that sits in the traffic path, inspects every packet, and can drop or reset sessions matching attack signatures; introduces latency and is a potential single point of failure.
- **IPsec** — A framework of protocols (IKE + ESP/AH) that encrypts and authenticates IP traffic between endpoints; the basis of site-to-site VPN.
- **IR playbook** — A pre-written, step-by-step procedure for a specific incident type (e.g., ransomware, credential stuffing, data exfiltration) that defines who does what, in what order, including pre-authorized containment actions — eliminates improvisation during an active incident.
- **IRSA (IAM Roles for Service Accounts)** — AWS's mechanism for giving EKS pods a cloud IAM identity via an OIDC token exchange, so pods assume a scoped IAM role without storing AWS access keys; the AWS equivalent of GCP Workload Identity Federation.
- **ISMS (Information Security Management System)** — The people, processes, and technology an organisation puts in place to manage information security risk systematically; ISO/IEC 27001:2022 is the internationally certifiable standard that defines what an ISMS must contain.
- **ISO 27001** — An internationally certifiable standard (ISO/IEC 27001:2022) for an Information Security Management System; Annex A lists 93 controls across Organisational, People, Physical, and Technological themes; certification requires audit by an accredited body and signals ISMS maturity to customers and regulators.
- **JIT (Just-In-Time access)** — A least-privilege pattern where elevated permissions are granted on demand for a bounded time window and automatically revoked on expiry, rather than held as standing access — the core mechanism of PAM and cloud PIM systems. S07.
- **Jitter** — Variation in packet inter-arrival delay; the key enemy of real-time audio/video; caused by queuing fluctuations; mitigated by LLQ and consistent low-latency paths.
- **Jumbo frames** — Ethernet frames with MTU up to 9,000 bytes (vs the standard 1,500), used on storage and high-performance networks to reduce per-packet overhead; requires consistent configuration across every switch in the path. (N30)
- **JWKS (JSON Web Key Set)** — A JSON document published by a token issuer (e.g. at `/.well-known/jwks.json`) that contains the public keys needed to verify JWTs the issuer has signed; services fetch JWKS to validate tokens without contacting the issuer on every request.
- **JWT (JSON Web Token)** — A compact, URL-safe token format (RFC 7519) consisting of a base64url-encoded header, payload (claims), and cryptographic signature; the payload carries identity assertions (sub, roles) and validity constraints (exp, aud) that the receiving service verifies using the issuer's public key — no session store required.
- **JWT algorithm confusion** — A class of vulnerabilities where a JWT validator trusts the alg field inside the token header rather than pinning the expected algorithm, allowing an attacker to forge tokens by supplying alg: none or switching from RS256 to HS256 — mitigated by hardcoding the accepted algorithm in validation logic.
- **KEK (Key Encryption Key)** — A long-lived symmetric key held inside a KMS or HSM boundary whose sole purpose is to encrypt and decrypt DEKs; the KEK itself never leaves the HSM, so an attacker who steals encrypted data and its encrypted DEK cannot read either without KMS access.
- **KEV (Known Exploited Vulnerabilities catalogue)** — CISA's continuously updated list of CVEs confirmed to be actively exploited in the wild; the most operationally urgent subset of the NVD — any KEV finding on an internet-facing or CDE asset should be treated as Critical regardless of its raw CVSS score.
- **Key custodian** — A named individual (or pair, for dual-control) responsible for the safekeeping, use, and lifecycle of a cryptographic key; PCI-DSS Req 3.7.1 requires formal appointment and documentation of custodians for manual key-management operations.
- **Key distribution problem** — The fundamental challenge of symmetric cryptography: if both parties need the same secret key to communicate securely, how do they share that key without it being intercepted? Solved in practice by asymmetric key exchange (e.g. ECDH in TLS).
- **Key lifecycle** — The defined states a cryptographic key passes through — generate, activate, use, deactivate, destroy — along with the policies governing transition between them, rotation schedule, and archive period for decrypting legacy ciphertext.
- **Key ring (GCP Cloud KMS)** — A GCP Cloud KMS grouping construct that bundles related CryptoKeys under a single IAM and regional boundary; the key ring's location determines where HSM operations execute and therefore which data-residency jurisdiction applies.
- **LACP** — Link Aggregation Control Protocol (IEEE 802.3ad / 802.1AX) — negotiates and maintains a bundle of physical links (a LAG) as one logical link; removes failed members automatically.
- **LAG** — Link Aggregation Group — a logical link composed of multiple physical links bonded together (via LACP); appears as a single interface to the network; also called port channel or bond.
- **Landing zone** — The pre-built network, IAM, and governance foundation that every application team inherits when provisioning cloud resources — the cloud equivalent of a shared data-center core fabric with enforced baseline controls.
- **Latency** — The time a packet takes to travel from source to destination; composed of propagation, transmission, queuing, and processing delay — the floor is set by physics (speed of light), the variable part is queuing, controlled by QoS. N53.
- **legal hold (litigation hold)** — A documented instruction, typically issued by legal counsel and executed by IT, to suspend normal data retention and deletion for specified data that may be evidence in a legal or regulatory proceeding.
- **LiME (Linux Memory Extractor)** — An open-source Linux kernel module that dumps physical RAM from a running system to a file or network socket without halting the host, enabling volatile memory capture during live IR without destroying evidence.
- **Link-local address** — IPv6 addresses in the fe80::/10 range that are automatically assigned to every IPv6-capable interface and are only routable on the directly connected link segment; always present even on IPv4-only networks.
- **LLQ (Low-Latency Queue)** — A strict-priority queue that empties before any other class is served; used to guarantee minimal latency for voice/EF traffic; must be capped (typically ≤ 33% of link BW) to avoid starving other classes.
- **LOCAL_PREF** — A BGP path attribute (0–4294967295, higher preferred) used inside one AS via iBGP to indicate which exit path is preferred; the primary dial for primary/backup interconnect design.
- **Log Bucket (GCP)** — A named, regionalized storage container inside GCP Cloud Logging with configurable retention period, optional CMEK encryption, and a lockable retention policy (Bucket Lock / WORM mode) that prevents deletion — the primary compliance-log archive construct in GCP.
- **log minimization** — The practice of stripping or masking sensitive field values (PAN, personal data) from log entries before SIEM ingest, preserving the structural event metadata needed for detection and audit while reducing GDPR/DPDP exposure.
- **Log Router (GCP)** — The Cloud Logging component that applies inclusion/exclusion filters to every log entry and routes matching entries to configured sinks (GCS, BigQuery, Pub/Sub, or another Log Bucket), enabling centralized archival, SIEM integration, and cost-reduction via exclusion filters.
- **Log4Shell (CVE-2021-44228)** — A critical remote-code-execution vulnerability (CVSS 10.0) in the Apache Log4j logging library (versions 2.0-beta9 to 2.14.1) that allowed an attacker to trigger JNDI lookups via crafted log messages, executing arbitrary code on the server — a canonical example of a CWPP-detectable vulnerability invisible to perimeter firewalls.
- **Longest prefix match** — The rule routers use when a packet matches multiple routing-table entries: the entry with the most specific (longest) prefix wins, allowing specific and default routes to coexist. (L3)
- **Loopback address** — 127.0.0.1 (and the 127.0.0.0/8 block): a special address that refers to the local machine itself; packets to it never leave the host's network stack.
- **LSDB (Link-State Database)** — The shared topology map that all OSPF routers in an area build by flooding Link-State Advertisements (LSAs); every router in the area holds an identical copy and runs SPF against it.
- **Luhn algorithm** — A checksum formula (ISO/IEC 7812-1) used to validate card PANs: digit doubling with mod-10 summation; DLP engines layer Luhn on top of a 16-digit regex to cut false positives by ~90 % before triggering an alert.
- **MAC address** (L2) — hardware address unique to a NIC; local-segment scope; rewritten at each hop. *N03, N05.*
- **MACsec (IEEE 802.1AE)** — Link-layer encryption standard that encrypts Ethernet frames between two directly connected devices; used to satisfy encryption-in-transit requirements on dedicated interconnect circuits without the overhead of a full IPsec tunnel.
- **Managed Identity (Azure)** — An Azure-managed non-human identity for workloads (VMs, Functions, App Service) that eliminates credential management by letting the Azure platform issue and rotate tokens; system-assigned ties to one resource, user-assigned is shared across resources.
- **management plane** — The network path used for administrative access (SSH, API calls, console, jump hosts, bastion); must be segmented and logged separately from the data plane.
- **MED (Multi-Exit Discriminator)** — A BGP path attribute sent to a neighbouring AS as a hint about which entry point is preferred; lower MED is preferred; used to influence inbound traffic on cloud dedicated interconnects.
- **MFA fatigue (push-bombing)** — An attack where an adversary who has already obtained a victim's password sends a flood of push-notification MFA prompts hoping the victim approves one to stop the noise; mitigated by number-matching, FIDO2 as the second factor, or account lockout after a threshold of denials.
- **MIB** — Management Information Base: the structured variable tree a network device exposes via SNMP; each variable has an OID (object identifier) path, e.g. ifHCOutOctets at 1.3.6.1.2.1.31.1.1.1.10.
- **Micro-segmentation** — Workload-level network policy enforcement — each VM, pod, or container gets its own firewall rules or identity-based policy, so east-west traffic is denied by default even between hosts on the same subnet.
- **MITRE ATT&CK** — A publicly maintained knowledge base of adversary tactics and techniques observed in real-world attacks (attack.mitre.org); used as a coverage map to audit which attack behaviors an organization can detect and where gaps exist.
- **MPLS (Multi-Protocol Label Switching)** — A telco/carrier technology that forwards packets using short numeric labels rather than IP routing at each hop, enabling private managed WAN services with defined QoS classes and topology without a dedicated physical circuit per site pair.
- **MSS** (L4) — Maximum Segment Size: largest TCP payload per segment; ≈ MTU − IP − TCP headers (1460 over a 1500 MTU). *N04.*
- **MSSP (Managed Security Service Provider)** — A third-party vendor that operates security functions — typically SOC monitoring, SIEM management, and incident response — on behalf of organisations that lack the headcount for an internal 24×7 team; common at mid-size FMCGs and smaller regulated entities.
- **mTLS** (security/L6) — mutual TLS: both client and server present certificates. *N21, S15.*
- **mtr (My TraceRoute)** — CLI tool combining `ping` and `traceroute`; shows per-hop latency, jitter (StDev), and loss in a single continuous report — the fastest way to locate which hop introduces jitter or loss. N53.
- **MTTD (Mean Time to Detect)** — The average elapsed time between an attacker or malware becoming active and the SOC raising an alert; the primary detection-effectiveness KPI; PCI-DSS and RBI guidance implies near-real-time detection for CDE events.
- **MTTR** — Mean Time to Repair: the average elapsed time from fault detection to full service restoration; a key SLA metric for FSI incident response and regulatory reporting.
- **MTU** (L2/L3) — Maximum Transmission Unit: largest payload a link carries in one frame (typically 1500 bytes Ethernet); tunnels lower it. *N04.*
- **NACL (Network ACL)** — AWS stateless subnet-level packet filter evaluated before Security Groups; requires explicit rules for both inbound and outbound directions including ephemeral reply ports (1024–65535).
- **NAPT (Network Address Port Translation)** — Synonym for PAT; the IETF term (RFC 2663) for the many-to-one NAT that maps private-IP:port pairs to a single public IP with distinct ports. N16.
- **NAT** (L3) — Network Address Translation: rewrites IPs (typically private↔ public). *N16.*
- **NAT Gateway** — Managed cloud service (AWS per-AZ; GCP regional Cloud NAT) that allows private-subnet instances to initiate outbound internet connections without being directly reachable from the internet.
- **NAT-T (NAT Traversal)** — Technique that encapsulates IPsec ESP packets in UDP (port 4500) so they survive NAT rewriting; required whenever an IPsec tunnel crosses a device that does PAT, such as a branch router or cloud NAT gateway. N16.
- **native VLAN** — The single VLAN on an 802.1Q trunk whose frames are sent *untagged*; misconfiguring it (e.g. leaving it as VLAN 1) enables VLAN hopping attacks.
- **NDP (Neighbor Discovery Protocol)** — ICMPv6-based protocol (RFC 4861) that replaces ARP in IPv6 networks; uses solicited-node multicast for address resolution, router discovery, prefix advertisement, and duplicate address detection.
- **NDR** — Network Detection and Response — a behavioral analytics platform that ingests flow telemetry or full-packet captures, baselines normal traffic, and detects anomalies such as lateral movement or slow exfiltration that signatures alone cannot catch.
- **NEG (Network Endpoint Group)** — GCP construct that adds individual IP:port endpoints — including on-premises hosts reachable via Interconnect — to a GCP load balancer backend service.
- **NetFlow** — Cisco-originated flow-export protocol (v5/v9); IPFIX is the IETF standard successor to v9. Routers export metadata about each IP flow (src/dst IP, port, byte count, duration) to a collector, revealing who is talking to whom and how much — without capturing payloads; the primary input for NDR and capacity monitoring.
- **Network ACL (NACL)** — (AWS/cloud) — a stateless packet filter applied at the subnet boundary in AWS VPCs; evaluated before Security Groups; both inbound and outbound rules must be written explicitly for bidirectional traffic.
- **Network address** — The first address in a subnet (all host bits set to 0); identifies the subnet itself in routing tables; not assignable to a host.
- **Network Connectivity Center (NCC)** — GCP's managed hub-and-spoke transit service that centralizes connectivity from VPCs, Cloud VPN tunnels, and Interconnect attachments into a single hub, replacing a custom transit-VPC pattern.
- **Network tag (GCP)** — A string label attached to a GCP Compute VM that routes and firewall rules can match on, allowing different VMs in the same VPC to follow different egress routes or firewall policies. N41, N42.
- **Next hop** — The IP address of the immediately adjacent router a host or router forwards a packet to; each device knows only one step, not the full path. (L3)
- **NGFW (next-generation firewall)** — (network security) — a stateful firewall that also inspects traffic at L7 to identify applications by behaviour rather than port alone; combines firewall, IPS, and application-ID functions in one device.
- **NIST CSF (Cybersecurity Framework)** — A voluntary US framework (v2.0, 2024) organised into six Functions — Govern, Identify, Protect, Detect, Respond, Recover — that provides an operating model for running a security programme; maps to ISO 27001 controls and is widely used for gap assessment and board-level reporting.
- **NIST SP 800-218 / SSDF (Secure Software Development Framework)** — US NIST guidance (SP 800-218) that defines practices for integrating security throughout the SDLC — threat modeling, code review, SAST/DAST, SBOM — referenced in US Executive Order 14028 and increasingly cited in FSI audit frameworks.
- **NMS** — Network Management System: the polling, alerting, and dashboarding platform (e.g. Zabbix, SolarWinds, PRTG) that aggregates SNMP counters and ICMP checks to track availability and utilization across the estate.
- **non-repudiation** — The security property that ensures an action or transaction can be proven to have originated from a specific party, and that party cannot credibly deny it — enforced via signed, tamper-evident audit logs; the violated property in STRIDE's Repudiation category.
- **non-transitive routing** — The property of VPC peering where a peered VPC will not forward traffic between two of its peers — traffic from A cannot reach C via B unless B runs a router (or managed transit construct) that re-originates the packet.
- **North-south traffic** — Network traffic that crosses a perimeter boundary — between an external client and an internal service; the traditional focus of perimeter firewalls, WAFs, and DDoS controls. Contrast east-west.
- **NSG / Security Group / firewall rule** (cloud) — cloud-native packet filters; models differ per cloud. *N42.*
- **NVD (National Vulnerability Database)** — The US NIST-managed authoritative registry of CVE records, including CVSS scores, affected software, and references; the primary source vulnerability scanners pull from to match detected software versions to known weaknesses.
- **NXDOMAIN** — DNS response code meaning the queried name does not exist; cached by resolvers for the SOA minimum TTL, so a mis-typed name stays 'not found' for minutes to hours.
- **OAuth2 (RFC 6749)** — A delegation framework (not an authentication protocol) that issues Access Tokens authorizing a client application to call an API on a user's behalf, with explicit, revocable, scope-limited consent.
- **OCSP (Online Certificate Status Protocol)** — Real-time protocol (RFC 6960) for checking whether a specific certificate has been revoked, queried against the issuing CA's OCSP responder.
- **OCSP Stapling** — Optimization where the TLS server pre-fetches and attaches a signed OCSP response to the handshake, giving clients revocation status with no added round-trip to the CA.
- **OIDC (OpenID Connect)** — An identity layer built on OAuth2 (OpenID Foundation, 2014) in which the IdP issues a signed JWT called an ID Token that identifies the authenticated user; the modern protocol for web and mobile app SSO.
- **order of volatility** — The collection priority for digital evidence ranked by how quickly it disappears: CPU registers and RAM first (lost on reboot), then swap/page file, active network state, disk, and finally logs — NIST SP 800-86 principle.
- **Org Policy (GCP)** — GCP's preventive guardrail system: constraints applied at the Organization, Folder, or Project level that block actions regardless of what IAM permissions a user holds (e.g., deny public IPs on VMs, restrict resource regions).
- **OSI model** — 7-layer reference model for networking. *N03.*
- **OSPF (OSPFv2)** — Open Shortest Path First — a link-state dynamic routing protocol (RFC 2328) used within an enterprise network; each router floods topology advertisements (LSAs), builds a shared Link-State Database, and runs Dijkstra's algorithm to compute the shortest path to every subnet.
- **out-of-band management** — A communications and administrative access path that is physically or logically separate from the primary network data plane, so that if the main network is compromised or down, responders can still access devices, run commands, and coordinate — critical during an active incident.
- **Overlay (SD-WAN)** — The IPsec-encrypted tunnels an SD-WAN controller builds on top of underlay circuits, abstracting the physical transport so policy can route traffic per-app across multiple paths. N33.
- **Oversubscription** — The ratio of total server-port bandwidth to available uplink bandwidth on a switch; a 3:1 ratio means 3 Gbps of potential server traffic compete for 1 Gbps of uplink — acceptable statistically, dangerous when unmeasured or ignored. (N30)
- **OWASP ASVS (Application Security Verification Standard)** — OWASP's level-by-level checklist of application security requirements that architects use to set verifiable security targets for a system — Level 1 covers basic hygiene, Level 2 covers most production systems, Level 3 covers high-assurance environments such as PCI-scoped banking applications.
- **OWASP Core Rule Set (CRS)** — The open-source managed rule set published by OWASP covering the OWASP Top 10 attack classes (SQLi, XSS, etc.); offered as a managed rule group by all major cloud WAF products. N25.
- **OWASP SAMM (Software Assurance Maturity Model)** — An open framework for measuring and improving a software security programme across five business functions (Governance, Design, Implementation, Verification, Operations) at four maturity levels; used by CISOs and architects to benchmark DevSecOps programmes and plan improvement roadmaps.
- **OWASP Top 10** — The Open Worldwide Application Security Project's ranked list of the most critical web application security risks, published every few years (2021 is current); used as a baseline requirement in PCI-DSS, RBI, and most FSI security design reviews.
- **Oxidized / RANCID** — Open-source tools (RANCID is the original; Oxidized the modern replacement) that SSH into network devices on a schedule, store the running config in version control, and diff each poll against the previous snapshot — an unexpected diff is an unscheduled change or a potential compromise.
- **PAC file (Proxy Auto-Config)** — A JavaScript file a browser fetches and executes per request to decide whether to send traffic via a proxy or directly; the enterprise standard for split-routing egress. (N23)
- **Packet loss** — The fraction of transmitted packets that never arrive at the receiver; causes TCP throughput collapse via slow-start and application-layer errors in UDP real-time workloads. N53.
- **Packet mirroring** — A cloud capability that copies raw packet bytes from selected VMs or subnets to a collector (IDS, PCAP store) without affecting the original traffic path; equivalent to a network TAP or SPAN port on-prem. N54.
- **PAM (Privileged Access Management)** — A system that vaults privileged credentials, brokers time-limited session checkout, records keystrokes/screens, and detects anomalous commands — replacing standing shared root passwords with auditable, revocable, just-in-time access.
- **PAN (Primary Account Number)** — The 16-digit card number printed on a payment card; the most tightly regulated piece of cardholder data under PCI-DSS — any system that stores, processes, or transmits a raw PAN is in PCI scope.
- **Parameterised query (prepared statement)** — A database query technique where user-supplied values are passed as separate typed parameters rather than concatenated into the SQL string, preventing SQL injection (OWASP A03) by ensuring the database engine never interprets user data as query syntax.
- **Passkey** — A synced FIDO2 credential whose private key is stored in the OS credential store (Apple Keychain, Google Password Manager, Windows Hello cloud) and replicated across a user's trusted devices, providing phishing-resistant authentication without the loss-of-device lockout risk of a hardware key.
- **PAT (Port Address Translation)** — Many-to-one NAT: multiple private hosts share one public IP, differentiated by source port; the form of NAT used at virtually every enterprise internet breakout and cloud NAT gateway. Also called NAPT or IP masquerade. N16.
- **PBKDF2 (Password-Based Key Derivation Function 2)** — An IETF standard (RFC 8018) that derives a cryptographic key from a password by applying a pseudorandom function (e.g. HMAC-SHA256) thousands of times with a salt, making brute-force and dictionary attacks computationally expensive.
- **PCI-DSS** — Payment Card Industry Data Security Standard: the 12-requirement framework (v4.0.1 current, superseded v4.0 on 2024-12-31) mandating network segmentation, access control, encryption, and audit logging for any system that stores, processes, or transmits cardholder data.
- **PE (Provider Edge)** — The carrier's router at the edge of the MPLS core that receives customer traffic from the CE, assigns (pushes) MPLS labels, and enforces the VPN and QoS policy the customer has contracted.
- **PFS (Perfect Forward Secrecy)** — A property of IKE Phase 2 re-keying: a fresh Diffie-Hellman exchange is performed each time, so compromising one session key exposes no past or future sessions.
- **Phishing-resistant MFA** — An authenticator category defined by NIST SP 800-63B and required by PCI-DSS v4.0 Req 8.4.2 for CDE admin access — specifically an MFA method (FIDO2, PIV) where the credential or code cannot be intercepted and replayed by a phishing relay because it is cryptographically bound to the legitimate origin.
- **PKCE (Proof Key for Code Exchange)** — An OAuth 2.0 extension (RFC 7636) that prevents authorization code interception in public clients (mobile apps, SPAs) by binding the token exchange to a secret verifier the client generates — required for mobile apps that cannot safely store a client secret.
- **PKI** (security) — Public Key Infrastructure: CAs, certificates, and trust chains. *N21, S10.*
- **Policing** — Enforcing a rate limit by dropping (or re-marking) packets that exceed the limit immediately; low memory cost but causes TCP retransmits; preferred at ingress/untrusted boundaries.
- **Policy Enforcement Point (PEP)** — The Zero Trust gate that proxies or blocks application sessions solely on the basis of the Policy Engine's decision; the resource sees only sessions the PE has approved.
- **Policy Engine (PE)** — The Zero Trust component that evaluates every access request against policy signals (identity, device posture, behavioral risk) and returns an allow/deny decision; it reasons but never touches data traffic.
- **PoP (Point of Presence)** — A CDN or network provider's physical edge location in a city or region; users are routed to the nearest PoP via anycast BGP. N25.
- **Port channel** — Cisco terminology for a LAG (link aggregation group) — the logical interface that bundles physical member links; also called a bond (Linux) or trunk (some vendors).
- **Private APN (Access Point Name)** — A carrier-provisioned mobile-network gateway that routes enterprise 4G/5G traffic through the carrier's private core rather than the public internet, providing a WAN-like private path over cellular.
- **Private DNS zone override** — A cloud-private DNS zone that resolves a managed-service hostname to a private endpoint IP rather than its public IP; required for private connectivity to actually be used — without it, VMs resolve to the public address and bypass the endpoint.
- **Private Hosted Zone (PHZ)** — (AWS) An AWS Route 53 DNS zone visible only to the VPCs it is explicitly associated with; used to serve private DNS names (e.g. RDS, internal services) in hybrid and cloud-only environments.
- **Private Service Access** — The older GCP pattern (now largely superseded by PSC) that allocated a /29 RFC 1918 range from your VPC and created a VPC peering to Google's managed-service VPC; limited by GCP's no-transitive-peering constraint.
- **Private Service Connect (PSC)** — GCP's mechanism for injecting a forwarding-rule-based private endpoint (with a VPC-local IP) into a consumer VPC, so managed services are reachable without a public IP or full VPC peering; supports both Google-managed services and producer-published custom services.
- **Private zone (DNS)** — A cloud DNS zone (GCP Cloud DNS / Route 53 Private Hosted Zone / Azure Private DNS Zone) whose records are served only to VPCs/VNets you explicitly attach or associate; external resolvers receive NXDOMAIN.
- **PrivateLink (AWS)** — AWS mechanism that injects an Elastic Network Interface (ENI) with a private IP into a consumer VPC subnet, routing traffic to an AWS-managed or partner service over AWS's internal fabric without traversing the internet.
- **privilege accumulation** — The gradual build-up of excess permissions on an account as roles are added (at each job change or project) but never removed; the silent outcome of a broken JML mover process and the most common source of PCI Req 7 findings. S07.
- **Propagation delay** — The fixed component of latency set by the physical distance light or electricity must travel through the medium; approximately 5 ms per 1,000 km in fibre — cannot be reduced by buying more bandwidth or tuning QoS. N53.
- **pseudonymisation** — Replacing directly identifying fields (name, national ID, mobile number) with a reversible surrogate token stored in a separate, access-controlled mapping table, so the main dataset cannot identify an individual without the key — the design pattern that simultaneously satisfies GDPR/DPDP right-to-erasure (delete the mapping) and RBI audit-log retention (keep the tokenised transaction record).
- **PSK (pre-shared key)** — A shared secret configured on both VPN peers to authenticate each other in IKE Phase 1; must be stored securely and rotated on a schedule (quarterly for PCI workloads).
- **PTR record** — DNS record type for reverse lookups — maps an IP address to a hostname; stored in the in-addr.arpa. zone with octets reversed; used by monitoring tools, mail servers, and audit logs.
- **public key pinning** — A client-side configuration that restricts which CA public key or certificate is accepted for a given service, regardless of what is in the system trust store; pinning the issuing CA's key (not the leaf cert) allows leaf rotation without breaking clients while still narrowing the blast radius of a misissued cert.
- **purple team** — A collaborative mode where red and blue team members work together during an exercise — the red team narrates each step so the blue team can immediately tune detections — accelerating the improvement cycle versus siloed red/blue operations.
- **purpose limitation** — The privacy principle (GDPR Article 5(1)(b), DPDP Section 4) that personal data may only be used for the specific, explicit purpose for which it was collected; an architectural constraint requiring that data pipelines carry purpose metadata so a mobile number collected for OTPs cannot be routed to a marketing campaign tool without fresh consent.
- **QoS (Quality of Service)** — The set of mechanisms — classification, queuing, policing, shaping — used to impose business-defined traffic priorities on a shared network link so critical flows (voice, trading) are not crowded out by bulk traffic.
- **QSA (Qualified Security Assessor)** — A PCI Security Standards Council-certified assessor company authorized to validate that a merchant or service provider meets PCI-DSS requirements; a QSA must agree that a tokenization implementation qualifies as a scope-reduction control before it is accepted as such in an audit.
- **Rainbow table** — A precomputed lookup table mapping common inputs (e.g. passwords) to their hash digests; used to crack unsalted password hashes instantly — defeated by adding a per-password random salt before hashing.
- **RASP (Runtime Application Self-Protection)** — A security agent embedded inside a running application that intercepts dangerous operations (SQL query execution, file reads, deserialisation) and blocks them in real time; unlike IAST it operates in production, not just in test, acting as a last line of defence when a vulnerability has not yet been patched.
- **Rate limiting** — An API gateway control that caps the number of requests a caller may make in a time window (e.g. 100 req/min per OAuth2 client_id), returning HTTP 429 on breach — the primary defence against credential stuffing, abuse, and accidental backend exhaustion.
- **RBAC** (security/IAM) — Role-Based Access Control: access via assigned roles. *S07.*
- **RBI Cyber Security Framework** — The Reserve Bank of India's Cyber Security Framework for Banks (2016, updated iteratively), which mandates continuous monitoring, network segmentation, incident reporting to CERT-In within defined windows, and data residency within India; legally binding for all scheduled commercial banks in India.
- **RBI IT Framework** — Reserve Bank of India Master Directions on IT Governance, Risk, Controls and Assurance Practices (2023): the regulatory mandate that governs network segmentation, change control, audit log retention, vendor access, and annual VAPT for scheduled commercial banks in India.
- **Re-wrapping** — The operation of decrypting a wrapped DEK with the old KEK and re-encrypting it with a new KEK after key rotation, without touching the bulk ciphertext — the efficiency mechanism that makes key rotation practical at scale.
- **Real-time phishing relay** — An attack where a phishing site acts as a transparent proxy, forwarding the victim's credentials and OTP codes to the legitimate site in real time before the code expires; defeats all time-based OTP methods (SMS, TOTP) but not FIDO2/WebAuthn, which refuse to sign for a non-matching origin.
- **Recursive resolver (full-service resolver)** — The DNS server that does all iterative work on behalf of a client — querying root, TLD, and authoritative servers in sequence, caching the result; your corporate DNS server or a cloud-provider DNS service plays this role.
- **red team** — A group of authorised ethical attackers who simulate real adversary techniques against an organisation's own systems to find control gaps the blue team and GRC may have missed; findings feed detection-rule improvements.
- **Refresh token** — A long-lived opaque credential issued alongside a short-lived access token; the client presents it to the authorization server to obtain a new access token without re-authenticating the user — revocation of the refresh token is the primary mechanism for ending a session.
- **Region (cloud)** — A named geographic cluster of cloud infrastructure (e.g. asia-south1 = Mumbai); the unit of data-residency compliance and latency zoning.
- **residual risk** — The risk that remains after all controls have been applied; the quantity the CISO formally accepts on behalf of the organisation and must be able to articulate to the board and auditor — 'we have no risk' is never a correct answer.
- **Reverse proxy** (L7) — sits in front of *servers*, terminating client connections; basis of LBs, WAFs, API gateways. *N24.*
- **RFC 1918** — The 1996 IETF standard reserving three IPv4 blocks (10/8, 172.16/12, 192.168/16) as private: routers on the public internet must discard packets with these addresses, so they are only meaningful inside an organization's network.
- **RFC 6598 / CGNAT range** — The 100.64.0.0/10 block defined by RFC 6598 for Carrier-Grade NAT; not RFC 1918 and not publicly routable — used by SD-WAN vendors for underlay management links and by cloud providers (e.g. AWS EKS pod CIDRs) to avoid exhausting private space.
- **right-to-audit** — A contractual clause that gives a customer or regulator the power to inspect a vendor's controls, evidence, and facilities — on defined notice — rather than relying solely on the vendor's self-assessment or third-party certifications; a mandatory clause for Tier-1 vendor relationships in FSI.
- **Risk** (security) — likelihood × impact; what security actually prioritizes. *S01.*
- **risk acceptance** — A formal, time-bound sign-off by a named risk owner acknowledging residual risk where a vulnerability cannot be remediated; must include compensating controls, a target remediation date, and the approver's name — not role — for audit purposes.
- **Risk appetite** — The amount of risk a board is willing to accept in pursuit of its strategy — a formally documented and annually ratified policy threshold that sits below the organization's risk capacity ceiling.
- **Risk capacity** — The ceiling of loss an organization can absorb before it ceases to operate; set by capital reserves, regulatory standing, and insurance — not a policy choice but a factual constraint.
- **risk register** — The documented inventory of identified risks, their likelihood and impact ratings, the controls treating them, and the residual risk remaining; the primary artefact GRC uses to demonstrate that risk management is active and evidence-based.
- **risk surface** — The total set of paths, interfaces, and conditions through which a threat could cause harm in a design; what a design review is actually walking.
- **Risk tolerance** — The acceptable variance band around the risk appetite for a given risk category — e.g. 'up to 3 open High findings in the CDE at any point' — narrower than appetite and used for day-to-day operational thresholds.
- **Risk treatment** — One of four formal dispositions applied to a risk item — Mitigate (add a control), Transfer (shift exposure via insurance or SLA), Accept (document and own the residual), or Avoid (stop the activity) — abbreviated as the 4 Ts.
- **risk walk** — A structured, ordered traversal of a design's security dimensions — scope, trust boundaries, identity, data flows, encryption, blast radius, detection, and compliance evidence — used to find gaps before they become incidents or audit findings.
- **Route 53 Resolver inbound endpoint** — (AWS) An ENI-based DNS endpoint inside a VPC that accepts DNS queries forwarded from on-premises networks, allowing on-prem hosts to resolve Route 53 Private Hosted Zone names.
- **router on a stick (ROAS)** — An inter-VLAN routing design where a single trunk link connects a switch to a router, which uses sub-interfaces — one per VLAN — to route between them; simple but the link is a bottleneck.
- **Routing table** — An ordered list of prefix/next-hop rules a host or router consults to decide where to forward each packet; processed by longest-prefix match. (L3)
- **ROV (Route Origin Validation)** — The act of checking a BGP advertisement against RPKI ROAs and dropping routes whose origin ASN or prefix doesn't match — the enforcement step of RPKI.
- **rpId (Relying Party ID)** — The domain string (e.g. meridian.example) embedded in a WebAuthn challenge and verified by the authenticator against the page origin before signing; the cryptographic mechanism that makes FIDO2 credentials origin-bound and therefore phishing-resistant.
- **RPKI (Resource Public Key Infrastructure)** — A cryptographic framework (RFC 6480) that ties ASNs to their authorised IP prefixes via ROAs; routers enforcing ROV reject advertisements that don't match, preventing BGP hijacks.
- **RPO (Recovery Point Objective)** — The maximum acceptable data loss measured in time (e.g. '15 minutes of transactions'); drives backup frequency and replication lag requirements in DR design.
- **RTO (Recovery Time Objective)** — The maximum acceptable elapsed time from an incident to full system restoration; sets the design requirement for DR infrastructure speed and is often mandated by regulation in FSI (e.g. RBI requires < 4 hours for core banking).
- **RTT (Round-Trip Time)** — The time from sending a packet to receiving its reply; what `ping` measures; approximately 2× one-way delay. N53.
- **Rule base** — (firewall) — the ordered list of permit/deny rules a firewall evaluates top-to-bottom, first-match-wins, against every packet; also called a rule set or policy.
- **SAD (Sensitive Authentication Data)** — Card-related security data — CVV/CVC, full magnetic-stripe content, PIN blocks — that PCI-DSS Req 3.3 forbids storing after transaction authorization, even in encrypted form.
- **SAML 2.0** — An OASIS XML-based SSO protocol where the IdP issues a signed XML assertion carried by the browser to the Service Provider; the dominant protocol for enterprise B2B and legacy SaaS federation.
- **SAMLResponse / SAML assertion** — The signed XML document an IdP issues and the browser carries to the Service Provider; contains the NameID, attribute statements (roles, groups), and a strict validity window (NotBefore / NotOnOrAfter).
- **SASE** — Secure Access Service Edge: a Gartner-coined architecture model converging network (SD-WAN, VPN) and security (SWG, CASB, ZTNA, FWaaS) into a cloud-delivered service, so security policy follows the user rather than the network perimeter. Referenced in N37, taught fully in S27.
- **SAST (Static Application Security Testing)** — Automated analysis of source code or compiled bytecode without executing the program, detecting insecure code patterns (SQL injection, hardcoded secrets, path traversal) at PR/commit time before the code ships.
- **SBOM (Software Bill of Materials)** — A machine-readable inventory (CycloneDX JSON or SPDX) of every component, version, and license in a software release — direct and transitive — enabling rapid CVE triage, license auditing, and supply-chain compliance evidence; stored alongside the container image or artefact.
- **SCA (Software Composition Analysis)** — Automated scanning of a project's dependency manifest (pom.xml, package.json, requirements.txt) — including transitive dependencies — to identify known CVEs and license-policy violations before the build ships.
- **SCP (Service Control Policy)** — AWS Organizations' deny-only guardrail: a policy attached to an OU or account that caps the maximum permissions any IAM entity in that scope can exercise, without granting any permissions itself.
- **Scrubbing center** — An upstream facility (ISP-operated or cloud-based) that receives diverted attack traffic via BGP, strips malicious packets, and returns clean traffic to the customer — the correct mitigation for volumetric DDoS that fills the customer's upstream link.
- **SD-WAN** (WAN) — software-defined WAN; cheaper, policy-driven branch connectivity. *N33.*
- **Security Command Center (SCC)** — GCP's managed security posture and threat-detection service; Standard tier (always-on) surfaces misconfiguration findings; Premium tier adds ML-based anomaly detection and threat-intelligence matching across all resources in a GCP organization.
- **Segmentation** (network/security) — dividing a network into isolated zones to limit blast radius. *N27.*
- **Segregation of duties** (security/process) — splitting a sensitive action across people so no one both performs and approves it (e.g. NetOps builds, the CAB approves a firewall change). *N02.*
- **Sensitive Data Protection** — GCP's managed DLP service (previously Cloud DLP) that scans GCS objects, BigQuery tables, or raw content for sensitive info-types (credit card numbers, Aadhaar, PII), returning findings with likelihood ratings and byte offsets for targeted redaction.
- **Service project** — (GCP Shared VPC) A GCP project attached to a host project that can deploy workloads (VMs, GKE, Cloud Run) into pre-allocated subnets without owning or modifying the network.
- **Service Provider (SP) / Relying Party (RP)** — An application that delegates authentication to an IdP, accepting its signed assertions or tokens rather than managing its own credential store.
- **Session affinity (sticky sessions)** — A load balancer mode that routes requests from the same client (identified by IP or cookie) to the same backend instance; a workaround for stateful apps that lack a shared session store.
- **shadow identity** — A cloud IAM user or credential created outside the organization's central directory (e.g. a local IAM user with a password instead of federated SSO), invisible to the identity governance process and not disabled when the person leaves.
- **Shaping** — Enforcing a rate limit by buffering over-rate packets and releasing them at the target rate; smoother than policing but adds latency; preferred for TCP bulk flows to avoid retransmit spirals.
- **Shared VPC** — (GCP) A VPC created in a host project and shared to one or more service projects, so a platform team owns the network plane (subnets, firewall rules, routing) while app teams own workloads running inside it.
- **SIEM (Security Information and Event Management)** — A platform that ingests, normalizes, and correlates log and telemetry data from across the environment to surface alerts and provide a searchable audit trail; the operational hub of a SOC.
- **Signed URL** — A time-limited, HMAC-signed URL generated by the origin application that a CDN PoP validates before serving a cached object, allowing private content to be cached at the edge without being publicly accessible. *N47.*
- **Sigstore** — An OpenSSF-hosted public-key infrastructure for software signing, comprising Fulcio (a short-lived certificate CA tied to OIDC identity), Rekor (an append-only transparency log of signing events), and the cosign CLI — enabling keyless artifact signing without managing long-lived private keys.
- **SLAAC (Stateless Address Autoconfiguration)** — IPv6 mechanism by which a host derives its own /64 address from the router-advertised prefix plus an Interface Identifier (EUI-64 or random), without a DHCP server.
- **SLSA (Supply chain Levels for Software Artifacts)** — An OpenSSF framework (originally Google) defining four hardening levels for build pipelines — from scripted builds with SBOMs (L1) through isolated ephemeral workers with signed provenance (L3) — giving auditors a comparable maturity grade for supply-chain security.
- **SNAT (Source NAT)** — Rewriting the source IP (and optionally source port) of an outgoing packet; the mechanism behind PAT/NAPT at egress points. Architects encounter SNAT exhaustion when a NAT device runs out of port numbers for concurrent flows. N16.
- **SNI (Server Name Indication)** — TLS extension sent in ClientHello carrying the requested hostname, allowing a single IP/load-balancer to select and present the correct certificate for each domain.
- **SNMP** — Simple Network Management Protocol: the polling and trap protocol (v1/v2c/v3) used by network management systems to read device counters and receive async fault alerts from routers, switches, and firewalls. SNMPv3 with auth+encryption is required in FSI.
- **SOAR (Security Orchestration, Automation and Response)** — A platform that automates repetitive SOC tasks (e.g. IP lookup, ticket creation, firewall block) and orchestrates multi-step incident-response playbooks, reducing analyst time-to-respond on known alert types.
- **SOC (Security Operations Centre)** — The function responsible for 24×7 monitoring, alert triage, detection engineering, threat intelligence, and incident response; measured by Mean Time to Detect (MTTD) and Mean Time to Respond (MTTR).
- **SOC 2 (Service Organisation Control 2)** — An AICPA auditing standard for service organisations (SaaS, cloud, BPO) that assesses controls against five Trust Service Criteria (Security, Availability, Processing Integrity, Confidentiality, Privacy); a Type II report covers a period of operation (6–12 months) and is an auditor attestation, not a certification.
- **socket** — The endpoint of a network connection identified by the pair IP:port; a TCP/UDP connection is uniquely described by the 4-tuple (src IP, src port, dst IP, dst port).
- **SPAN port** — Switch Port Analyzer — a switch feature that copies traffic from one or more ports/VLANs to a designated monitoring port, used to feed passive IDS or NDR sensors without interrupting the original traffic flow.
- **SPDX (Software Package Data Exchange)** — A Linux Foundation SBOM standard (ISO/IEC 5962:2021) that describes packages, files, and snippets with license expressions; one of the two dominant SBOM formats alongside CycloneDX.
- **Spine-leaf** — A two-stage Clos switching fabric where every leaf switch connects to every spine switch, giving every server pair exactly two hops and consistent east-west latency; the standard topology for modern data centers. (N30)
- **Split tunneling** — A VPN/ZTNA configuration where only traffic destined for private ranges routes through the tunnel while internet traffic exits locally; improves performance but creates a compliance concern for PCI-scoped users who are simultaneously connected to untrusted internet and the CDE.
- **Split-horizon DNS (split-view DNS)** — A DNS configuration that returns different answers for the same name depending on the source of the query — typically a private IP to internal clients and a public IP to external ones; used to hide internal topology.
- **SSE (Security Service Edge)** — The cloud-delivered security-services stack — ZTNA + SWG + CASB + FWaaS — that enforces policy for users reaching the internet and SaaS; a subset of SASE that excludes the SD-WAN transport layer.
- **SSL-VPN** — A remote-access VPN that tunnels traffic over TLS (port 443) rather than raw IPsec, improving traversal through hotel and corporate firewalls; the dominant client VPN form since the 2010s.
- **SSRF (Server-Side Request Forgery)** — A vulnerability where an attacker tricks a server into making HTTP requests on their behalf — to internal services, cloud metadata endpoints, or private VPC addresses — by supplying a crafted URL as user input; especially dangerous in cloud environments where the metadata endpoint at 169.254.169.254 returns instance credentials.
- **Standard Contractual Clauses (SCCs)** — EU Commission-approved contract templates (updated 2021) that legally authorise transfers of personal data from the EU/EEA to countries without an adequacy decision; the most commonly used GDPR transfer mechanism — architects must ensure their data-processing agreements include the correct SCC modules before routing EU personal data to a cloud region outside the EEA.
- **Stateless firewall (packet filter)** — (network/L3–L4) — a firewall that evaluates each packet in isolation with no memory of prior packets; requires explicit rules for return traffic in both directions.
- **Sticky session (session affinity)** — An LB configuration that binds a client's requests to the same backend — typically via a cookie or IP hash — to preserve server-local state; trades HA for state locality.
- **STP (Spanning Tree Protocol)** — IEEE 802.1D protocol that prevents L2 loops in switched networks by blocking redundant paths; runs per VLAN in PVST+ and can cause outages if root bridge election is misconfigured.
- **STRIDE** — A threat taxonomy (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege) used to enumerate threats systematically against each component and trust-boundary crossing in a design.
- **Subject Alternative Name (SAN)** — An X.509 certificate extension listing all hostnames, IP addresses, or email addresses the certificate covers; the authoritative field for hostname validation — the legacy Common Name (CN) field is no longer accepted by modern browsers.
- **Subnet** (L3) — a sub-division of an IP network sharing a routing prefix. *N09.*
- **Supernet** — A large address block that is subdivided into smaller subnets; e.g. Meridian Bank's 10.0.0.0/8 is the enterprise supernet, carved into /16 blocks per site.
- **SVI (Switched Virtual Interface)** — A virtual L3 interface configured on a Layer 3 switch for a specific VLAN; acts as the default gateway for all hosts in that VLAN and is where inter-VLAN routing happens on-switch without an external router.
- **SWG (Secure Web Gateway)** — A cloud-hosted or on-prem forward proxy with URL categorisation, DLP, and TLS inspection; the managed-service evolution of Squid-style proxies. (N23)
- **SYN cookies** — A TCP stack mechanism (RFC 4987) that encodes connection state into the sequence number instead of a table entry, making a server immune to moderate SYN floods without external mitigation.
- **SYN flood** — A DDoS protocol attack that sends millions of TCP SYN packets with no follow-up ACK, filling the server's half-open connection table until it rejects legitimate connections.
- **tabletop exercise** — A facilitated walkthrough of an incident scenario with IR team members and decision-makers, testing the IR plan and playbooks without touching production systems — the primary way FSI organizations fulfil regulatory requirements to test incident response.
- **threat modeling** — A structured design-time practice that identifies trust boundaries, enumerates threats (typically via STRIDE), rates each by risk, and produces a prioritized list of mitigations and documented accepted risks.
- **Throughput** — The actual data rate delivered end-to-end over a measurement window; always ≤ bandwidth and reduced by packet loss, TCP flow control, and protocol overhead. N53.
- **TIME_WAIT** — TCP state after a graceful connection close where the socket waits 2×MSL (roughly 60 s) before being freed; at high connection rates this can exhaust the ephemeral port range and cause new connections to fail.
- **TLS** (L6) — Transport Layer Security: encrypts and authenticates sessions. *N21.*
- **TLS inspection (SSL bumping)** — A proxy technique that terminates the client TLS session, re-signs the server certificate with an internal CA, and re-originates a new TLS connection outbound — allowing the proxy to read and filter HTTPS payloads; requires the internal CA to be trusted on every managed device. (N23)
- **TLS re-encryption** — A reverse-proxy configuration where TLS is terminated at the proxy (decrypted), then a new TLS session is opened to the backend — providing end-to-end encryption while still allowing the proxy to inspect and route the request; required in PCI-DSS CDE and many FSI designs.
- **TLS termination** — The point in a network path where a TLS session ends and plaintext is handed to the next component; the termination point determines who can inspect traffic and is a key PCI-DSS design decision.
- **Token bucket** — A rate-limiting algorithm that allows burst traffic up to a bucket capacity, then refills at a steady rate — suitable for APIs where short bursts are legitimate but sustained overload must be capped.
- **token introspection** — An OAuth2 endpoint (RFC 7662) a resource server calls to ask the authorization server whether a given access token is currently active; enables real-time revocation checks, unlike purely self-validating JWTs.
- **token vault** — The isolated, PCI-scoped database that holds the mapping between a surrogate token and the encrypted real PAN; the only system permitted to detokenize, with every call audited.
- **Tokenisation** — Replacing a sensitive value (e.g. a card PAN) with a non-sensitive surrogate token at the earliest point of entry, so downstream systems never handle the original value and fall outside PCI-DSS scope.
- **TOTP (Time-based One-Time Password)** — A second-factor code (RFC 6238) generated by combining a shared secret with the current Unix time, producing a 6-digit value that changes every 30 seconds; used by authenticator apps (e.g. Google Authenticator) and mandated by PCI-DSS Req 8.4 for privileged-access MFA.
- **TPRM (Third-Party Risk Management)** — The end-to-end lifecycle — discover, assess, contract, monitor, offboard — by which an organisation evaluates and controls the security risk posed by external vendors, software components, and service integrators; mandated by PCI-DSS Req 12.8 and RBI outsourcing guidelines.
- **traceroute** (tool/L3) — maps the routers on a path by sending packets with increasing TTL and reading the ICMP "time exceeded" replies. *N06.*
- **Traffic Mirroring** — AWS's per-ENI packet-mirroring feature: copies raw traffic from individual Elastic Network Interfaces to a Network Load Balancer or target ENI for IDS/PCAP analysis. N54.
- **Traffic selector (IPsec)** — The src/dst IP range pair that defines which packets are encrypted by a given IPsec SA; must match on both peers or the tunnel establishes but no traffic flows.
- **Transit Gateway (TGW)** — AWS's managed hub-and-spoke transit service that connects VPCs and on-prem networks through a single routing hub with configurable route tables and per-attachment propagation policies.
- **Transparent proxy** — A proxy that intercepts traffic at the network layer without client configuration; functionally equivalent to a forward proxy but invisible to the application. (N23)
- **trunk port** — A switch port that carries multiple VLANs simultaneously on one physical link, using 802.1Q tags to identify which VLAN each frame belongs to; connects switches to switches or switches to routers/L3 switches.
- **trust boundary** — A line in a design where the security zone or trust level changes (e.g. internet → DMZ, or cloud VPC → on-prem DC); the primary organizing concept for a design review.
- **Trust Service Criteria (TSC)** — The five assessment dimensions defined by the AICPA for SOC 2: Security (mandatory), Availability, Processing Integrity, Confidentiality, and Privacy; an organisation chooses which criteria to include in its SOC 2 scope based on customer commitments.
- **trust store** — The bundle of trusted root CA certificates pre-installed in an OS, browser, JVM, or container image that acts as the ultimate anchor against which certificate chains are validated; each runtime environment maintains its own (OS, JVM, Firefox, container base image) and they must all be updated when an enterprise adds a new internal root CA.
- **TTL** (L3) — Time To Live: a hop counter in the IP header; decremented by each router, the packet is dropped at 0 (basis of traceroute). *N06.*
- **TTP (Tactics, Techniques, Procedures)** — The three layers of attacker behavior described in MITRE ATT&CK: Tactics (the goal, e.g. lateral movement), Techniques (the method, e.g. T1021 Remote Services), and Procedures (the specific implementation); TTPs persist longer than IoCs and are the preferred basis for durable detection rules.
- **Tunnel mode (IPsec)** — IPsec mode that wraps the entire original IP packet (including its inner headers) inside a new IP packet — the VPN gateways are the outer IPs, the inner addresses are private. Used for site-to-site VPN.
- **ULA (Unique Local Address)** — IPv6 addresses in the fc00::/7 range (commonly fd00::/8) defined by RFC 4193; privately routable within an organization but not advertised to the internet — the IPv6 equivalent of RFC 1918.
- **Underlay (SD-WAN)** — The physical circuits an SD-WAN overlay rides: MPLS, broadband, 4G/5G — the underlay just moves bits; the overlay adds policy. N33.
- **VAPT** — Vulnerability Assessment and Penetration Testing: the combined practice of scanning for known weaknesses (VA) and actively attempting exploitation (PT) to find what automated scans miss; mandated annually by RBI for critical banking systems.
- **vendor risk tiering** — The practice of classifying third-party vendors into risk tiers (Tier 1 = critical / data-access; Tier 2 = hosted data, no transactions; Tier 3 = tooling only) to calibrate the depth of due diligence — on-site audit for Tier 1, self-assessment questionnaire for Tier 3 — so GRC effort matches actual risk.
- **VEX (Vulnerability Exploitability eXchange)** — A structured document (CISA/OWASP standard, embedded in CycloneDX 1.4+) in which a vendor or team asserts whether a known CVE in their SBOM is actually exploitable in their specific deployment context, reducing triage noise from scanner false positives.
- **VIP** — Virtual IP — an IP address shared by multiple physical devices (e.g. the HSRP/VRRP virtual IP); no single device owns it permanently; ownership transfers on failover.
- **Virtual Interface (VIF)** — AWS Direct Connect's logical partition of a physical port — a private VIF connects to one VPC, a transit VIF connects to a Direct Connect Gateway for multi-VPC/multi-region reach.
- **Virtual IP (VIP)** — A single IP address presented to clients by a load balancer, masking the pool of backend servers behind it.
- **virtual patch** — A WAF rule, IPS signature, or network-level block that prevents exploitation of a vulnerability without modifying the vulnerable software binary — used to buy time in change-frozen or OT systems where the underlying code cannot be immediately patched.
- **Virtual WAN Hub (vWAN)** — (Azure: TODO) Azure's managed transit hub construct, offered in plain and Secured (Azure Firewall embedded) variants; analogous to GCP NCC or AWS Transit Gateway.
- **VLAN** (L2) — Virtual LAN: logical L2 segmentation over shared switches. *N15.*
- **VLAN attachment (Interconnect Attachment)** — GCP's logical sub-circuit object that maps an 802.1Q VLAN on a Dedicated or Partner Interconnect port to a Cloud Router and VPC, analogous to AWS's Virtual Interface (VIF) or Azure's ExpressRoute peering.
- **VLAN hopping** — An attack where a device on an access port sends double-tagged 802.1Q frames to reach a VLAN it should not have access to, typically exploiting a misconfigured native VLAN.
- **VLSM** — Variable Length Subnet Masking: allocating subnets with different prefix lengths (e.g. /28 for a small DMZ, /22 for a large server farm) to match actual host counts and avoid wasting address space.
- **VPC / VNet** (cloud L3) — Virtual Private Cloud (GCP/AWS) / Virtual Network (Azure): your private cloud network. *N39.*
- **VPC endpoint** — A private, regional connection from a VPC to a cloud provider's service (e.g. AWS VPC Interface/Gateway Endpoint, GCP Private Service Connect endpoint) that routes traffic inside the cloud network, bypassing NAT gateways and avoiding public-internet egress charges.
- **VPC Flow Logs** — Per-flow connection-metadata records (src/dst IP, port, protocol, bytes, disposition) exported by cloud hypervisors at the subnet or NIC level; used for security auditing, anomaly detection, and compliance evidence. Available in GCP and AWS. N54.
- **VPC peering** — A private, non-transitive L3 connection between two VPCs that routes traffic over the provider backbone without NAT or internet traversal; each pair of VPCs that must communicate requires its own bilateral connection.
- **VPC Reachability Analyzer** — AWS tool that performs a config-level simulation of a packet path between two endpoints in a VPC, identifying the routing or security-group rule that would block it, without sending real traffic.
- **VPC Service Controls (VPC SC)** — GCP-specific API-level security perimeter that restricts which identities and resources can access GCP services (e.g. Cloud Storage, BigQuery), preventing data exfiltration even when IAM permissions are present; no exact AWS equivalent.
- **VPN concentrator** — The gateway device (firewall or dedicated appliance) at the enterprise edge that terminates remote-access VPN tunnels, authenticates clients, and routes them to internal resources; a high-value attack target because it is internet-facing and processes unauthenticated packets.
- **VRRP** — Virtual Router Redundancy Protocol (RFC 5798) — open-standard FHRP; functionally equivalent to HSRP but multi-vendor; elects a Master and one or more Backups.
- **WAF** (L7/security) — Web Application Firewall: inspects HTTP for attacks. *N25.*
- **WebAuthn** — The W3C Level 3 browser API (web.dev/passkeys) through which a web application requests a FIDO2 authentication or registration ceremony; the browser enforces that the relying-party ID (rpId) in the challenge matches the current page origin before calling the authenticator.
- **well-known ports** — Port numbers 0–1023, reserved by IANA for standard services (e.g. 22 SSH, 53 DNS, 80 HTTP, 443 HTTPS); binding to them on a server requires root/administrator privilege.
- **Wildcard mask** — The bitwise inverse of a subnet mask used in Cisco ACLs; 0 bits mean 'must match,' 1 bits mean 'ignore' — convert to CIDR by subtracting each octet from 255.
- **Workforce Identity Federation (GCP)** — A GCP IAM mechanism that accepts third-party OIDC or SAML tokens from an external IdP (e.g. Entra ID, Okta) and maps them to GCP IAM principals, eliminating the need for Google accounts for enterprise staff.
- **Workload Identity (GCP)** — The GCP mechanism by which a Kubernetes Service Account or other workload identity is mapped to a GCP Service Account, allowing pods to call Google APIs without exporting a service account key file.
- **Workload Identity Federation** — A GCP mechanism (and general cloud pattern) that lets workloads outside GCP — on-prem services, AWS EC2s, CI pipelines — exchange short-lived cloud-provider credentials for GCP access tokens without storing a service account key file, eliminating a major credential-leakage risk.
- **WPAD (Web Proxy Auto-Discovery)** — A protocol that lets clients automatically find the PAC file URL via a DNS lookup for wpad.<domain> or DHCP option 252; convenient but can be hijacked if DNS is not locked down. (N23)
- **X-Forwarded-For** — An HTTP header added by a proxy or load balancer carrying the original client IP address, since the TCP source IP seen by the backend is the LB's own IP; standardised in RFC 7239 as the Forwarded header.
- **X.509** — The ITU-T/IETF standard (RFC 5280) defining the format of a digital certificate — the encoding all browsers, OSes, and services speak when validating TLS, code signatures, or client authentication.
- **XKS (External Key Store, AWS)** — AWS KMS's External Key Store feature that lets a KMS key reference a customer-operated external key manager (on-prem HSM or third-party service); every AWS KMS call is proxied to the external store, so the key never enters AWS.
- **Zero Trust** (security) — never trust by location; verify identity and context on every request. *S26, N27.*
- **Zone (cloud)** — An independent failure domain within a cloud region — separate power, cooling, and building — used to spread resources for high availability without the cost of multi-region.
- **ZT maturity model** — A staged framework (notably CISA's five-pillar model covering identity, devices, networks, applications, and data) for scoring current Zero Trust capability and planning incremental improvement — the tool that turns 'we're doing Zero Trust' into a measurable roadmap.
- **ZTNA** — Zero Trust Network Access: an architecture that grants per-application proxied sessions based on identity and device posture, rather than assigning a client IP on the corporate network — eliminating lateral-movement risk inherent in client VPN.
- **ZTP (Zero-Touch Provisioning)** — A deployment method where a network device (SD-WAN CPE, switch) dials home to a controller on first boot, downloads its full configuration, and becomes production-ready without on-site engineer intervention. N33.
